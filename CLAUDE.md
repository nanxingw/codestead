# CLAUDE.md

## 项目是什么

Codestead 是一款星露谷（Stardew Valley）风格的 2D 像素农场游戏，同时也是 Claude Code 多会话的「候车室」：当 AI 在干活时，你在一个连贯、低压的游戏世界里劳作，而不是跳进碎片化的信息流；屏幕左上角始终保有对所有 agent 会话状态的感知；游戏里偶尔出现的 NPC 任务由后端 AI 根据你当前的真实工作即时生成，帮你梳理思路、做出决策。

## 初衷：为什么做这个

在 AI 时代的编码范式下，许多程序员出现了类似 ADHD 的症状：等待 AI 交付结果时无事可做，只能刷手机、刷推特。这种碎片化信息的轰炸，加上工作状态的来回切换，对大脑伤害很大。

Codestead 用一个完整的游戏世界承接这些等待间隙：

- **替代信息流**：等待时你在同一个连贯世界里种田、建造，而不是被算法推送撕碎注意力；
- **保持感知**：左上角 HUD 让你随时知道哪些会话在工作、哪些在等你输入，到点就能回去；
- **引回思考**：AI 生成的 NPC 任务与你正在做的工作相关，把注意力轻轻引回工作本身的思考与决策。

## 理想形态

### 1. 一个真正好玩的农场游戏（游戏第一）

星露谷风格的 2D 像素农场：开辟农田、种菜、浇水、收割、出售农作物、建造房屋、提升农场等级。内容要丰富，可玩性一定要高——它必须先是一个独立成立的好游戏，而不是披着游戏皮的仪表盘。

### 2. 会话状态 HUD（感知，不干预）

- 左上角显示本机所有 Claude Code 会话及其状态：working（工作中）/ blocked（等待用户输入）/ done（已完成未查看）/ idle（空闲）；
- **只展示，不干预**：不打断玩法、不强制暂停、不绑定游戏经济。玩家自己决定何时切回工作；
- 技术方案参考 herdr（https://github.com/ogulcancelik/herdr）：前台进程识别 + 终端输出检测，零配置可用。Codestead 优先使用 Claude Code hooks（SessionStart / Stop / Notification 等）获得可靠的语义状态，进程/输出检测作为兜底。

### 3. AI 即时生成的关卡（NPC 任务/对话）

- 后端常驻一个 headless Claude Code（`claude -p`），读取各会话当前的工作上下文，即时生成村民 NPC 的任务与对话；
- 形态是决策题与思路梳理：村民找你聊你正在做的架构取舍、重构边界、接口设计等，帮助你树立思路、思考决策；
- 你的回答会被记录为思考笔记，未来可回填给对应的 Claude 会话；完成任务获得游戏内奖励（金币/道具/经验）；
- 关卡是思考的辅助，绝不是打扰：出现频率克制，可以整体关闭。

## 设计原则（按优先级）

1. **游戏第一**：可玩性优先于一切监控/工具属性，玩法本身要经得起单独评价；
2. **感知不干预**：会话状态只展示，永远不强制打断玩家；
3. **本地与隐私优先**：工作内容（transcript、关卡上下文）只在本机处理，不上传任何服务器；
4. **开源产品标准**：文档、安装体验、跨平台兼容都按可发布的开源产品要求来做。

## 架构蓝图（规划，尚未实现）

pnpm workspaces monorepo：

- `packages/game` — 游戏客户端：Phaser 3 + TypeScript + Vite，跑在浏览器里；
- `packages/daemon` — 本地守护进程：Node.js + TypeScript，
  - 会话监控：Claude Code hooks + 进程检测 → 会话状态机，
  - WebSocket 服务（localhost）向游戏推送状态，
  - 关卡生成：调用 headless Claude Code，基于会话上下文生成 NPC 任务；
- `packages/shared` — daemon 与 game 共享的协议与类型定义。

数据流：`Claude Code 会话 → (hooks / 进程检测) → daemon → WebSocket → 游戏 HUD 与 NPC 系统`

## 开发约定

- TypeScript everywhere；代码标识符与注释用英文，面向用户的文档中文优先（发布前补英文版）；
- 与项目所有者的交流使用简体中文；
- 美术资产必须使用许可允许的素材（CC0 / 明确授权）——做的是星露谷「风格」，绝不能使用星露谷本体素材；
- 默认分支 `main`，远程 `https://github.com/nanxingw/codestead`。

## 术语表

- **会话（session）**：一个运行中的 Claude Code 实例；
- **四态**：working / blocked（等待用户输入）/ done（完成未查看）/ idle；
- **关卡（quest）**：由后端 Claude Code 即时生成的、与当前工作相关的 NPC 任务/对话。
