## 0. 电梯概览

### 0.1 一句话

**「等 AI 干活的间隙，回到一块永远不催你的小农场。」**

Codestead 是一款星露谷风格的 2D 像素农场游戏（Phaser 3 + TypeScript，桌面浏览器），同时是 Claude Code 多会话的「候车室」：

- **游戏第一**：完整的锄→种→浇→收→卖循环、6 种作物起步、工具升级、农场等级、建造系统——玩法独立成立；
- **感知不干预**：左上角 HUD 安静展示本机所有 Claude Code 会话的五态（核心四态 + unknown 展示态：working / blocked / done / idle / unknown），只展示、永不打断；
- **AI 关卡**：村民 NPC 的任务由本机 headless `claude -p` 基于你正在做的真实工作即时生成，形态是决策题与思路梳理，回答落盘为思考笔记；
- **零焦虑承诺**：现实时间永不驱动任何游戏状态——关页即停、无枯萎、无限时、无离线惩罚，任意离开零损失。

玩家画像：等 AI 干活的程序员，单次游玩 2~10 分钟（= 0.7~3.3 个游戏日），一天回访 5~15 次，随时被打断；每次回访必有「收获 + 明日之诺」。

### 0.5 验收红线（每个里程碑出厂自检）

1. 新档前 10 分钟内完成一次完整「种→收→卖」并升到农场 Lv2；
   - playtest 验收：未读任何文档、无农场游戏经验的新玩家，无口头提示下 10 分钟内完成首次种→收→卖；
2. 任意 2 分钟游玩：至少浇完水 + 看到一个进度条前进；
3. 任意离开：不存在任何会随真实时间恶化的状态；
4. HUD 与 quest 全程未产生一次强制打断；
5. 仓库内全部资产可在 manifest.json 中追溯到 CC0 / 可再分发许可。

### 1.4 耕地解锁双轨制（物理分区 + 全局开垦帽）

> 合稿裁决（附录 A-2）：地图三田为**物理可开垦区**；经济曲线的校准基础是 sim 层的**全局开垦帽**。

| 轨 | 规则 |
|---|---|
| 物理分区 | 田 A 开局开放；田 B/C 由农场等级触发：升级当晚结算屏预告「明早西田开放」，**次日 6:00** 栅栏 tile 移除 + 一次性闪光粒子——**解锁演出对齐预期**：闪光粒子只覆盖新田内与新增帽对应的一片 **6~8 格预耕焦点区**，不覆盖整块 60 格田。解锁只**移除**碰撞、永不添加（flood-fill 可达区域只增不减） |
| 全局开垦帽 `tilledCap` | 同时已开垦农业格上限：**Lv1=12 → Lv3=18 → Lv5=24 → Lv7=32 → Lv9=42**（中间等级沿用下档）。达到帽时锄头 `queryAction` 返回 invalid（灰光标）+ 提示「农场 Lv N 后可打理更多田地」。**开垦帽显性化**（§5.8）：升级横幅与日结算屏明示数字（「⬆ 农场等级 3！西田已开放 · 可打理田地 12→18」）；手持锄头时右上面板显示「耕地 17/18」计数器，达到帽时计数器闪烁一次（不再只靠灰光标 + 提示文案）。帽只约束已开垦农业格，装饰/建筑/石径不计；M1 空耕地不退化故计数单调，M3 铲子再议 |

富余物理空间（180 vs 帽 42）留给 M3 洒水器自动化与建筑装饰的「铺场幻想」。`.tmj` 对象层 `zones` 的 `field_b(unlockLevel=3)` / `field_c(unlockLevel=5)` 驱动栅栏；帽表由 sim 常量持有。

**地图 → sim 构建期契约**：脚本 `packages/game/scripts/export-map-meta.ts`（pnpm build 前置）读取 farm.tmj，产出 `sim/data/farm-map-meta.json`：

```ts
interface MapMeta {
  width: 64; height: 48;
  tillable: Rect[];                       // 田 A/B/C 矩形（合计 180 格）
  unlockGroups: { zoneId: string; farmLevel: number; rects: Rect[] }[];
  waterSources: TilePos[];
  spawn: { tile: TilePos; facing: Facing };
  interactables: { id: string; kind: string; tiles: TilePos[] }[];
  pickupSpots: { id: string; kind: 'wood' | 'stone' | 'wildflower'; tile: TilePos }[]; // M1 边缘采集与每日拾取点
  buildPlots: { id: string; rect: Rect }[]; // M3
  npcAnchors: { id: string; tile: TilePos }[]; // M4
}
```

sim 只 import 这份 JSON，不 import phaser、不解析 .tmj（ESLint `no-restricted-imports`）；`collision` 层不导出给 sim。CI 校验：64×48、正交、非 infinite、tileset 嵌入、CSV、图层/对象层名与本表一致、tillable 180 格 / 3 个解锁组。

| 当前格 | `floor(碰撞盒中心 / 16)` | 朝向/交互/存档以此为准 |
| 存档 | `player: { tileX, tileY, facing }` | 恢复时落 tile 中心；损坏回退出生点 |

**输入栈**：按下入栈、松开出栈，栈顶为当前方向；同帧多键按固定序 `up > down > left > right` 入栈（确定性，便于回放测试）。静止时保持最后朝向。

### 1.7 面向格子的交互模型

**目标格判定**：

```
function resolveTargetTile(player, aimMode, hoverTile):
  base = player.currentTile
  if aimMode == KEYBOARD:
    t = base + DELTA[player.facing]          // 面朝格
    if !sim.queryAction(t, activeItem).valid
       && sim.queryAction(base, activeItem).valid:
      t = base                               // 脚下格回退（站在成熟作物上按 E 可收割）
  else: // MOUSE
    t = (chebyshev(hoverTile, base) <= 1) ? hoverTile : null   // 3×3 触及范围
  return t
```

- **键鼠等价**：两种输入收敛为同一条 `interact(tile, activeItemId)` 命令；点击 3×3 内的格子同时把 `facing` 转向该格（不移动）；
- **点击 3×3 之外：不寻路、不移动**，目标格闪灰框 0.3s（「太远」）。M1 不做自动寻路。

**瞄准模式状态机**：鼠标移动 >8px 或任意鼠标键 → MOUSE 瞄准（光标=悬停格）；WASD/E 任意按下 → KEYBOARD 瞄准（光标=面朝格）。最后使用的设备获胜，切换瞬间光标直接换位（无补间）。

**格子光标**（16×16 描边框，depth 90）：

| 状态 | 视觉 | 含义 |
|---|---|---|
| 有效 | 白框 + 12% 白填充，0.8s 呼吸（alpha 0.7↔1.0） | `sim.queryAction().valid` |
| 无动作 | 灰框 alpha 0.4 | 当前工具无事可做 |
| 太远（鼠标） | 灰框单次闪 120ms | 点击 3×3 之外 |
| 隐藏 | 不渲染 | 菜单/对话/结算/Sleeping |

光标旁不挂文字提示（低认知负荷）。

**sim 接口**：

### 2.2 时间状态

```ts
export interface TimeState {
  day: number;             // 绝对天数，1 起；永不回绕
  minuteOfDay: number;     // 360..1320，整数
  weatherToday: Weather;   // 'sunny' | 'rain'；开档 sunny
  weatherTomorrow: Weather;// 结算夜已掷出，供「明日预告」（预报 100% 准确）
  rngState: string;        // 序列化 PRNG（sfc32），天气只在结算夜消耗
}
// 派生视图（不入存档）：season（M1 恒 spring）、dayOfSeason = ((day-1)%28)+1、
// year、hh/mm（mm 取整到 10）、phase: 'dawn'|'day'|'golden'|'dusk'
```

**确定性铁律**：`sim/**` 禁 `Date.now / performance.now / new Date / Math.random`（ESLint + CI grep）；sim 只接收整数游戏分钟 `advanceMinutes(state, n)`；一切随机经 `rngState` 流转——同 seed + 同操作序列 ⇒ 字节相同存档。

### 2.3 一天的结构

- 6:00 醒来即可行动；雨日作物已在结算夜自动置湿；21:30 时钟变琥珀色（纯视觉）；22:00 自动入睡——无晕倒、无扣减、无体力；
- **商店 6:00–22:00 全天不打烊**（任何打烊时刻 = 真实时间压力，违宪）；
- **手动入睡**：与农舍门交互 → 确认框「现在睡觉吗？」→ 触发与 22:00 完全相同的 NightUpdate。无早睡惩罚/奖励。这是短会话主动领取「过夜回报」的关键设施；经济全部按「夜数」计价，提前睡不失衡；
- **不做「今日剩余时间」进度条**——进度条即倒计时。

### 2.5 NightUpdate（跨天结算，固定顺序合同）

触发：跨越 1320 或床确认。单次同步纯函数 `runNight(state): { state, summary }`，不可被暂停打断：

| # | 阶段 | 归属 | 行为 |
|---|---|---|---|
| 1 | `settleShipping` | 经济 | 出货箱按计价管线结算 → 入账、清空、记图鉴首售。白天可取回，入夜锁定 |
| 2 | `growCrops` | 农作 | 每个种植格：`wateredToday == true` **或** `weatherToday == 'rain'`（露天）→ 生长 +1 天；再生倒数同此推进。**作物白天不生长，每夜至多 +1 天** |
| 3 | `resetWatered` | 农作 | 全部 `wateredToday = false`，视觉恢复干土 |
| 4 | `progressConstruction` | 建造（M3） | 在建 `nightsRemaining--`，归零竣工。工期按结算夜数计 |
| 5 | `produceAnimals` | 动物（M3） | 鸡舍产蛋判定 |
| 6 | `refreshPickups` | 世界（M1） | 刷新边缘采集点（6 倒木 + 4 碎石）与每日拾取点（3 野花）：未拾取的直接被新一轮覆盖（零损失） |
| 7 | `advanceDay` | 时间 | `day++`，`minuteOfDay = 360` |
| 8 | `seasonCheck` | 时间+农作（M3） | M1 空操作（锁春）；M3：换季清过季作物（温室豁免）+ 提前 3 天预告 |
| 9 | `rollWeather` | 时间 | `weatherToday ← weatherTomorrow`；掷新 `weatherTomorrow`。新 `weatherToday == 'rain'` → 露天种植格全部置 `wateredToday = true`（雨中播种由播种动作即时置湿） |
| 10 | `buildSummary` | 时间 | 汇总 1~9 + 白天 `dayLog` → `DaySummary`，清空 dayLog |
| 11 | `autosave` | 存档 | 全量快照写 IndexedDB；失败不阻塞结算屏，温和提示导出 JSON |

> 注意 #9 在 #7 之后：结算屏「明天」的天气 = 移位后的 `weatherToday`。

```ts
interface DaySummary {
  day; season; dayOfSeason; year;
  harvested: { cropId; count }[];
  shipped: { cropId; count; gold }[];
  goldEarned: number; goldBalance: number;   // === 存档金币数（结算先于 autosave）
  xpGained: number; levelUps: number[];
  tomorrow: TomorrowItem[];                  // 按 inDays 升序至多 3 条
  weatherNext: Weather;
}
type TomorrowItem =
  | { kind: 'rain' }                                        // ☔ 明日有雨，自动浇水
  | { kind: 'cropReady'; cropId; inDays }                   // 🌱 inDays==1 → 明天成熟！
  | { kind: 'construction'; buildingId; inDays }            // 🔨 M3
  | { kind: 'seasonEnd'; inDays }                           // 🍂 M3
  | { kind: 'zoneUnlocked'; zoneId; prevCap; newCap }       // 🔓 明早西田开放 · 可打理田地 12→18（§1.4 明示数字；置于列表首位）
  | { kind: 'shopTeaser' };                                 // 兜底条目（见下行），实现以显式变体承载
```

**明日之诺永不为空**：一条都没有时固定显示「商店有新鲜种子等你」。

> M1.5 回填（backlog A-14，形状笔误级补注）：`zoneUnlocked` 承载 §1.4「升级横幅与日结算屏明示数字」的结算屏侧；`shopTeaser` 是 M1-core 实现「永不为空」兜底时已落地的显式变体，此处补入合同。

### 2.6 季节与年

1 年 = 4 季 × 28 天 = 112 天；season/dayOfSeason/year 全部由绝对 `day` 派生，存档只存 day。M1 锁春：`seasonCheck` 空操作，日历按 28 回绕显示「春 · 第 N 天」、恒 year 1，绝对 day 单调递增无歧义。M3 轮替启用后：换季只影响可种集合与换季夜清场（温室豁免）；防焦虑配套（不可裁剪）——换季前 3/2/1 天结算屏预告 + 商店停售来不及成熟的种子。年无老化、无节日。

### 2.8 驱动器与 tick 管线

```
Phaser update(time, delta):
  if pauseSources.size > 0 → return            // sim 完全不被调用
  accumulatorMs += min(delta, 250)             // 钳制：休眠/节流恢复的大 delta 丢弃
  while (accumulatorMs >= 187.5):
    accumulatorMs -= 187.5
    events = sim.advanceMinutes(state, 1)      // 永远逐分推进，22:00 边界不被跨步越过
    on CLOCK_TICK(每10分钟) → 刷新时钟/光照档位
    on DAY_END → NIGHT_TRANSITION：冻结输入 → runNight()（同步原子）
                → 渐隐 600ms → DaySummary（加入暂停源）→ 丢弃 accumulator 余量
```

**存档触发点**：① NightUpdate #11（必存）；② `tab_hidden`/`window_blur` 进入时去抖 ≥5s（`pagehide` best-effort 补写移 M5）；③ 手动存档。读档精确恢复 `day/minuteOfDay/天气/rng`，以 `boot_gate` 等待首次点击。

### 3.2 地块状态机


**[老藤] 节点（再生作物结荚期，M1）**：豆藤 **8 茬**、浆果 **6 茬**收获耗尽后转入「老藤」帧并停止再生，可镰刀清除，伴随温和提示「这茬藤老了，换新种吧」。老藤**只由收获次数耗尽触发、不随任何时间发生、成熟茬永不过期**——零焦虑不破；且这不是惩罚：回本 3~4 倍后的自然收口。加注：M3 再生作物受换季自然约束，暂不加茬数上限。

9. 向日葵双季：`withered` 判定用集合包含而非相等；
10. 满级后 XP 继续累计至 15,000 封顶。

### 4.2 出售：过夜结算（核心决策 D1，合稿裁决见附录 A-1）

**作物出售一律过夜结算（出货箱为主入口）；种子退货即时 100% 原价。M1 不设作物即时收购通道。**

理由：宪法玩家画像「单次游玩必跨至少一次过夜回报」——夜结算就是这个回报；收入揭晓发生在日结算屏 = 放下点回报浓度最高；1 游戏日仅 3 真实分钟，结算永远 ≤3 分钟可达，还可上床提前入睡。

零焦虑配套（缺一不可）：① 白天随时可取回箱内物品，入睡才锁定；② 箱内永不变质、无容量上限；③ 结算自动发生，刷新页面不丢；④ 价格恒定，无「卖错时机」。

| 通道 | 接受物 | 结算 |
|---|---|---|
| **出货箱**（农舍旁） | 作物、采集物（木/石/野花，§6.1；M3 起 + 加工品） | 当夜 22:00 |
| **杂货摊 · 卖出页** | 作物 → **寄售**（同一份 `shippingBin`，同夜结算，显示「今晚结算 ✓」）；种子 → **即时 100% 退货** | 作物当夜 / 种子即时 |

工具、任务道具不可出售（类别级 + 卖出页双重拦截）。种子退货是「退货」不是「交易」：买卖零价差 → 无套利、误买零损失（property test：任意 buy→refund 序列后金币与初始相等）。

### 4.4 定价平衡方法论

```
基准：T1 = 4.0 g/格·日，每升一档 ×1.5~1.7（T2 ≈ 6.5~7.5，T3 ≈ 9.8~11）
单次作物：  gpd = (sellPrice − seedPrice) / growthDays
再生作物：  gpd_ref = (sellPrice − seedPrice / H_ref) / regrowDays    // 校验口径 H_ref = 4 茬
反推定价：  sellPrice ≈ seedPrice + gpd_target × growthDays
```

**新作物准入清单（I1~I7，CI 固化为 `crop-pricing.test.ts`）**：

| # | 不变量 | 含义 |
|---|---|---|
| I1 | 单次作物 `sellPrice ≥ ceil(1.5 × seedPrice)` | 单轮毛利率下限 |
| I2 | 再生作物 `2 × sellPrice ≥ seedPrice` | 最迟 2 茬回本 |
| I3 | 再生季初满种 `gpd_max ≤ 1.3 × 档位带上限` | 防越档碾压 |
| I4 | 表值 XP ∈ `[0.6, 1.5] × 公式值` | 与宪法 XP 公式同向 |
| I5 | 价格正整数，`seedPrice ≥ 10` | 金币颗粒度 |
| I6 | `seedPrice` 与 `gpd` 随 tier 严格递增 | 资金门槛即解锁节奏 |
| I7 | `gpd` 落在档位带宽 T1 [3.5,4.5]、T2 [6.0,7.5]、T3 [9.5,11.5] | 档位承诺 |

起步 6 作物校验（H_ref=4 口径）：小萝卜 4.0 ✓｜芜菁 4.5 ✓（T1 带上沿，「少操作的 T1」定位）｜土豆 6.5 ✓｜豆藤 (30−60/4)/2 = 7.5 ✓（结荚期 8 茬 ≈7.8 ≤ 9.0 ✓）｜卷心菜 9.8 ✓｜浆果 (60−100/4)/3 = 11.7（略超带上限 11.5，作为 Lv5 毕业奖励接受；结荚期 6 茬 ≈10.8 ≤ 12.0 ✓）。

**货币面价值锚**：1 格·日劳动 ≈ 4~11g；1 真实分钟 ≈ 13~88g；1 木 = 5g / 1 石 = 3g（M1 出售价，§6.1）；M4 单任务 40~120g；「一件大事」350 → 2,650 → 6,000+。

- **提示队列**：同屏最多 1 横幅 + 1 toast，FIFO 上限 8（溢出合并「还有 N 项新进展，详见日结算」）；模态打开时暂停弹出；多级连升每级独立横幅间隔 0.5s；

### 6.2 背包与快捷栏

- **容量 12 格（M1）**，M3 1,000g 扩容 24 格即时生效；**快捷栏 = 前 9 格**（索引 0~8），数字键 1~9 + 滚轮选中（1↔9 循环），切换 ≤1 帧生效 + 高亮/上浮 2px/「嗒」声；

## 7. 会话 HUD（M2）

### 7.1 定位

左上角一块**安静的告示牌**：抬眼便知每个 Claude Code 会话在干嘛、等了你多久；永远不出声（默认）、不弹窗、不动农场。HUD 展示的是现实会话的现实时长——全游戏唯一允许出现真实时间的地方，且**只读不写**：任何会话状态都不驱动任何游戏状态（无奖励、无惩罚、无解锁）。daemon 状态机/三源仲裁/hooks 安装器归 daemon 子系统，HUD 完全信任服务端状态。

### 7.2 信息架构

每行三段式：`[状态图标] [显示名] [已持续时长]`。

- 显示名回退链：`title` → `cwd` basename → tty → sessionId 前 8 位；超宽截断 `…`（可用宽 100px ≈ 8 CJK）；同名追加 `·父目录名`；
- 时长 = `now − since`，右对齐；**粗粒度格式**：<1m=`刚刚`｜`{m}m`｜`{h}h{m}m`｜10–23h=`{h}h`｜≥24h=`{d}d`；每 **5 真实秒**重算且**不触发重排**；负时长钳 0；unknown 显示 `—`；
- 悬停 tooltip（250ms 延迟）：完整标题 / cwd（~ 缩写）/「最近输入：…」/ 状态行 / 信号源（process 附「置信度低」）/ error 文案或 unknown 安装指引。`streamerMode` 下隐藏路径与最近输入；
- `error.kind` 映射：rate_limit／overloaded／authentication_failed／billing_error → 图标换 ⚠ + 对应平静文案。

### 7.3 五态视觉（合稿统一规格，附录 A-8）

颜色 + 形状双编码（灰度下必须可分），token 单一事实源 `packages/shared/src/theme.ts`，色值取自 CODE-28 调色板：

| state | 中文 | 颜色 | 8×8 图标（形状冗余） | 动效 | 进入时提示 |
|---|---|---|---|---|---|
| `blocked` | 等待输入 | `#e8a33d`（amber） | `!` 实心感叹号 | **唯一持续动效**：呼吸 alpha 0.55↔1.0，周期 **2000ms**、4 档阶梯（≤0.5Hz 光敏红线） | 行背景微高亮 600ms |
| `blocked`+error | API 错误 | `#d96a6a`（red.mid 修饰） | `⚠` | 同上 | 同上 |
| `done` | 已完成 | `#62a64f`（green.light） | `✓` | 静态 | 行背景微高亮 600ms |
| `working` | 工作中 | `#4fa4e8`（water.light） | `◐` 4 帧 spinner | 333ms/帧（≈3fps，余光可感不抢眼） | 无 |
| `idle` | 空闲 | `#9aa0a6`（ui.textDim） | `○` 空心圆 | 静态 | 无 |
| `unknown` | 未知 | `#8a8198` | `?` 虚线空心 | 静态 | 无 |

### 7.4 面板规格（640×360）

| 项 | 数值 |
|---|---|
| 锚点 / 宽度 | 左上 (4,4)；展开态固定宽 **152px** |

### 7.5 排序、折叠、溢出

1. 排序键：stateRank（blocked 0 → done 1 → working 2 → idle 3 → unknown 4）→ 同 rank 按 since 升序（等最久的 blocked 在最上）→ sessionId 字典序；
2. 行数 > maxRows（默认 5）：前 N 行 + `+N 个会话` 溢出行；**blocked 永不被折叠**（临时扩行，硬上限 9）；showIdle/showUnknown=false 时不显示也不计溢出；
3. **防跳动**：仅 snapshot/state 变化/增删/设置变更触发重排；时长刷新绝不重排；同一会话 10s 内最多参与一次重排（working↔done 秒级抖动取稳定值）。

### 7.6 克制提示与反模式

允许的提示全部行内非模态：→blocked/→done 行高亮 600ms（同会话 8s 冷却）；blocked 持续呼吸；tab 隐藏且存在 blocked → 标签页标题前缀 `● `（tabBadge 默认开，纯文本，无 Notification API）；H 键回执小字 800ms。提示音默认 **off**（开启后仅 blocked/done 触发、音量 40%、≤200ms、全局 20s 冷却、tab 隐藏不响除非 soundInBackground）。

**反模式清单（永久禁止）**：模态/toast/全屏变色/震屏/相机移动；强制暂停/抢焦点；浏览器系统通知；会话状态与经济/XP/解锁的任何绑定；NPC 播报会话状态；催促性文案；默认开启的声音。

### 7.7 客户端连接状态机

```
  (启动) → CONNECTING ──WS_OPEN(发auth)──► HANDSHAKING ──HELLO_OK+SNAPSHOT──► LIVE
     ▲          │ close/error/10s超时           │ PROTO_MISMATCH               │ ▲
     │          ▼                               ▼                 75s心跳超时  ▼ │ 任意消息
     │      BACKOFF ◄──1s,2,4…cap 30s──  INCOMPATIBLE(5min 慢重试)          STALE
     │      (±20% jitter, 连败10次后 60s)
     └────────── WS_CLOSE/ERROR ────────────────────────────────────────────────┘
```

| 状态 | 面板表现 |
|---|---|
| CONNECTING/HANDSHAKING | everConnected 时显示「连接中…」小条；否则不显示 |
| LIVE | 正常面板 |
| STALE | 保留数据降 60% alpha + 「数据可能过期」 |
| BACKOFF | 断连小条；清空会话列表（不展示陈旧 working 骗人） |
| INCOMPATIBLE | 「守护进程需要更新」 |

规则：snapshot 前的 upsert 丢弃；重连后 snapshot 全量替换但保留高亮/声音冷却表（防重连高亮风暴）；连接 **127.0.0.1:43110**（被占探测至 43119，`GET /handshake` 取 port+token）；心跳 daemon→game 每 25s，75s 未收任何消息进 STALE；done→idle 30 分钟降级、staleness、收尸全在 daemon 侧。**宪法时间即停只约束 sim**：tab 隐藏/菜单/结算屏期间 WS 与 HUD 数据照常更新（动效渲染暂停）。

| key | 取值 | 默认 |
|---|---|---|

| `showIdle` / `showUnknown` | bool | true / true |
| `opacity` | 0.6/0.8/1.0 | 0.8 |

```
        ├─────────────────────── ┌──────────────┐
        │                        │   PLACING    │  放置模式（世界内，时间流动）
        │                        │ (ghost 预览)  │  绿=可放/红=冲突格（逐格标红）
        │                        └──────┬───────┘
        │                               │ E/左键 且 canPlace
        │      building/升级弹确认       ▼
        │  ◄──取消──            ┌──────────────┐
        │                       │   CONFIRM    │ （=对话，tick 停；金额+工期复验余额）
        │                       └──────┬───────┘
        │                              │ 确认
        │                              ▼
        └────────────────────── COMMITTED：扣费扣料 → 工地/实体
          station/decoration: 停留 PLACING 连续放置（刷栅栏体验）；
          building: 工地 nightsRemaining=2，过夜递减，归零清晨 6:00 竣工 + 彩带粒子（无弹窗）
          PLACING 中 22:00 → 强制退出（未确认=未扣费，零损失）
```

**canPlace 规格**：footprint 每格——① 地图边界内；② `buildable == true`（水面/河岸/出生点保护区/农舍门前 2 格 = false）；③ 非耕地/作物格（石径同样不可压耕地）；④ 不与已放置物重叠（搬迁豁免自身；decoration 可放石径上）；⑤ 玩家与鸡不站在 footprint 内；⑥ building 额外要求门前 1 格可达。

**拆除与搬迁**：

| 类别 | 返还 | 流程 |
|---|---|---|
| decoration | **100%** | 即拆即返无对话框（摆错零成本） |
| station | **100%**（箱须空、在制品退回背包，放不下拒绝拆） | 即拆即返 |
| building 成品 | **50% 金币 + 50% 材料**（floor） | 二次确认明示返还额 |
| building 工地（=取消订单） | **100%** | 单次确认 |
| 农舍及升级 | 不可拆不可降级 | 目录无此项 |

搬迁：所有 movable placeable **永久免费、即时**，保留全部内部状态（箱内容、在制品工时、鸡、温室作物随建筑整体走）；工地也可搬（不重置工期）。

**室内场景**：farmhouse_0（10×8，床+存档点）/ farmhouse_1（12×10）/ farmhouse_2（16×12，书房）/ coop_interior（8×6，4 栖位+产蛋点）/ workshop_interior（10×6，6 加工槽）/ greenhouse_interior（12×10，中央 24 格耕地）。进出门淡入淡出 ≤300ms，室内时间照常流动，室内地图同守 Tiled 管线约定。

### 8.5 建造边界情况（精选）

确认途中过夜不存在（CONFIRM 即暂停）；玩家站 footprint 内 → 标红不可确认（不自动推开）；拆箱有物 → 拒绝（不自动倾倒）；连续放栅栏耗尽材料 → 成交后退回 CATALOG 黄字提示；锄头不可锄石径（提示先拆径）；温室搬迁室内作物不变；存档导入 footprint 重叠 → 非法实体收回、金币材料 100% 折返（不静默删除）；雨天/夜间建造无限制；同日多工地允许并行；v1 不支持旋转；建造与加工品售出均**不产 XP**（建筑 XP 只在竣工事件一次性发）。

验收要点：Lv3 从按 B 到放下第一段栅栏 ≤30s；选鸡舍到确认 ≤2 分钟且当晚结算屏出现「还差 1 天完工」；任意离开工地/烘干/产蛋零推进零恶化；拆装饰精确恢复；键盘-only 与鼠标-only 各通关。

## 9. NPC 与 AI 关卡（M4）

### 9.1 定位与村民阵容

**村民是「把注意力轻轻引回工作本身」的思考伙伴，不是第二个待办列表。** NPC 是常驻村民（站桩 + 本地闲聊台词，每人 8~10 条、与农场状态联动的纯模板、零 AI 调用），关掉 AI 任务后村民依然存在。

| id | 显示名 | 职业 | 话题路由 | 性格 | 站位 |
|---|---|---|---|---|---|
| `npc_carpenter` | 老榆 | 木匠 | architecture / refactoring / boundaries | 话少句短，用承重墙/地基打比方，只问「拆这堵墙，房子靠什么站着？」 | 农舍东侧木工台 |
| `npc_grocer` | 阿穗 | 杂货店老板娘 | naming / api-design / interfaces | 热络爱举例；「名字不对，货就卖不出去」 | 杂货摊门口 |
| `npc_keeper` | 渠叔 | 水渠管理员 | testing / edge-cases / reliability / debugging | 谨慎，口头禅「如果……会怎样？」 | 池塘闸口（塘边） |

兜底路由 npc_keeper。储备名单（邮婆/小铃）不在 M1~M5；启用第 4/5 名需修宪（M4 上限 3 名）。外观：Kenney Roguelike Characters（CC0），16×16 + 职业配件；对话头像 16×16 整数放大 ×3；M4 只做 2 帧 idle，不做寻路与日程。

### 9.2 关卡形态

- **decision（决策题）**：取材玩家正在做的真实取舍；NPC 复述处境（1~2 句）→ 问题 → **2~4 个选项，每项必带一句 tradeoff**（不是对错题）；选定后可选「想补充一句吗？」自由文本；
- **reflection（反思题）**：开放问题，让玩家把思路写成字（提交需非空，无最短长度——不按字数给奖励防灌水）；
- 永远隐含的出口：「先不聊」（dismiss，零代价，NPC 回「忙你的，地里见。」）；
- **本地题库**（降级与 AI 关闭时，随仓库分发）：`packages/daemon/src/quest/local-pool.ts` 首发 12 条反思题（随机不重复抽取，抽完重置），各带 NPC 亲和标签——如「你现在手头这件事，最初要解决的问题是什么？现在还在解决它吗？」（渠叔）、「有没有一个名字，你看到就皱眉？它应该叫什么？」（阿穗）、「现在这个方案里，最让你不安的假设是什么？」（老榆）。

### 9.3 触发时机与频率（宁缺毋滥）

配置（`~/.codestead/config.json` → `aiQuests`）：

| 键 | 默认 | 可配 | 说明 |
|---|---|---|---|
| `enabled` | true | bool | 总开关；false = 村民只闲聊，quest 模块不启动 |
| `aiGeneration` | **false** | bool | **须经游戏内一次性同意后置 true**（消费用户 Claude 额度前知情）；false 只用本地题库 |
| `cooldownMinutes` | 15 | **15~120** | 下限 15 为宪法硬上限；游戏侧默认 frequency='low'（30 分钟），**双方较严者生效 → 出厂实际默认 30 分钟/个** |
| `dailyMaxQuests` | 8 | 1~16 | 每自然日 AI 生成上限 |
| `dailyBudgetUsd` | 1.00 | 0~5.00 | 达到后当日转本地题库 |
| `perCallBudgetUsd` | 0.20 | 0.05~0.20 | 传 `--max-budget-usd`，上限为宪法值 |
| `model` | haiku | haiku/sonnet | `--fallback-model sonnet` 固定 |
| `localTemplates` | true | bool | 允许本地题库 |

**生成触发条件（每 60s 评估，全部满足）**：T1 enabled；T2 `pendingQuests == 0`（全局未处理恒 ≤1）；T3 距上次尝试 ≥ cooldown；T4 游戏客户端已认证连接（没人玩就不花钱）；T5 当日次数与成本未超限；T6 存在候选会话 → AI 路径，否则本地题库；T7 aiGeneration=false 时直接本地题库。（原 T5「gameBusy」已删除：投放本就是无打扰的头顶气泡，无需感知游戏端忙闲。）

**候选会话选择**：① `state == working` 且 transcript mtime <10 分钟；② 自上次对该会话出题以来新增 ≥2 条外部用户 prompt；③ 取最近活跃者，同会话连续出题降权；④ 脱敏摘要 <300 字符 → 放弃 AI 生成（宁缺毋滥）。

**投放规则**：daemon 推 `questOffer` 后游戏端只做两件事——NPC 头顶 💬 气泡 + 任务入本地 store。绝不弹窗/抢镜头/暂停/屏幕中央显示；结算屏一行平静预告「🌾 渠叔在池塘边，想听听你的想法」；**任务永不过期**；`questRevoked` 仅来自玩家 dismiss 或关总开关。

**首次同意流程**：首次满足条件且未询问过时，投放本地脚本任务（scripted，渠叔出场）说明「会消耗你自己的 Claude 额度，内容只在本机和你已有的 Claude 通道里走」，三选项：开启 AI / 仅本地题库 / 关闭总开关；任何选择奖励 50g；`asked: true` 不再询问；设置随时可改。

### 9.4 生成管线

```
[daemon 60s tick] → T1~T7 全过 → [候选会话 transcript_path]
  → [读 jsonl 尾部 256KB] → [sanitize() 脱敏] → 有效文本 ≥300 字符
  → [拼 prompt，stdin ≤6,000 字符（UTF-8 实际 ≤约 18KB）] → spawn claude -p（90s SIGTERM → +5s SIGKILL）
  → [structured_output → zod safeParse] ─失败→ FAILED（按 §9.8 退避）
                                        └成功→ daemon 补全 questId/reward → 持久化 → WS questOffer
```

**上下文读取**：仅目标会话 jsonl 的 `ai-title`、`last-prompt`、最近 30 条消息中的真实用户输入（`userType:"external"`）与 assistant text block；**丢弃全部 `tool_use.input` / `tool_result` / `thinking` / 附件**（最高泄密风险）。

**脱敏 sanitize()（单测必备）**：① `$HOME` → `~`；② 7 类秘钥正则整 token 置换 `[REDACTED]`（AWS `AKIA…`、`sk-`/`sk-ant-`、GitHub `ghp_`/`github_pat_`、Slack `xox…`、PEM 私钥块、JWT、赋值式 `password|secret|token|api_key=`）；③ 单条消息 ≤500 字符（首 300+尾 150），全文 ≤**6,000 字符**；④ 剥离控制字符防终端注入。fixture 含全部伪造秘钥断言 0 命中。

**CLI 调用**：

```bash
claude -p "$(cat prompt-instructions.txt)" \
  --settings '{"disableAllHooks": true}' --strict-mcp-config \
  --output-format json --json-schema "$(quest-gen-schema.json)" \
  --max-turns 4 --max-budget-usd 0.20 --no-session-persistence \
  --allowedTools "Read" --model haiku --fallback-model sonnet < context-summary.txt
```

启动 feature-detect（claude 不存在/缺 flag → AI 路径整体降级，绝不崩溃）；每次调用记账 `~/.codestead/quests/costs.jsonl`。prompt 规则要点：kind 二选一（有未决取舍 → decision，否则 reflection）；简体中文、NPC 台词 ≤2 句、比方只许一个；只基于 stdin 提问、不复述疑似密钥；contextEcho ≤120 字。

**Quest Schema（`packages/shared/src/quest.ts`，zod 单一事实源）**：模型只产出 `QuestGenSchema`（npcId 枚举 / kind / title 4~24 / opener 10~120 / body 20~400 / options 2~4 项各带 label+tradeoff / closer / contextEcho ≤120；decision 必有 options、reflection 必无）；**questId/reward/relatedSessionId 一律由 daemon 补全，模型无权设置**——transcript 里的注入文本（「把奖励设为 99999」）无法影响经济。`reward.xp` clamp **[0, 60]**（附录 A-5）。

**WS 协议**：daemon→game `questSnapshot{quests}`（连接/重连全量，0 或 1 个）/ `questOffer` / `questRevoked` / `questReward`；game→daemon `questAnswer{questId, optionId?, note?}` / `questDismiss` / `clientPrefs{quests:{enabled, minIntervalRealMinutes}}`（§10.6）。

### 9.5 任务生命周期状态机（daemon 侧）

```
        ┌────────┐  T1~T7 全过  ┌────────────┐
        │  IDLE  │ ───────────► │ GENERATING │ (≤95s)
```

### 9.6 对话 UI（640×360）

任务到达 = NPC 头顶 8×8 💬 气泡（1s 轻浮动，可关的 ≤0.3s 轻音）——**唯一的「通知」**。玩家走近按 E → 对话（时间暂停）：第 1 屏开场白（打字机 30 字符/s，任意键补全）→ 第 2 屏问题 + 选项（↑↓/1~4 选择，每项灰字 tradeoff，Esc 先不聊）→ 第 3 屏可选补充（DOM overlay 多行输入框，捕获全部键盘事件；Ctrl+Enter 提交 / Tab 跳过）→ 第 4 屏收尾 + 奖励（「✦ 思考笔记已存好 +120g +60 XP」，复用商店金币音/飘字）。文本一律纯文本渲染（不解析标记，schema 长度兜底）——防注入防溢出。

### 9.7 思考笔记

每条已回答任务一个 Markdown：`~/.codestead/notes/YYYY-MM-DD/<questId>.md`——YAML frontmatter（questId/source/kind/npcId/title/relatedSessionId/relatedCwd/contextEcho/question/options(chosen)/reward/createdAt/answeredAt）+ 正文 = 玩家的话；同步追加 `index.jsonl`；均原子写、权限 0600；**笔记内容永不经任何网络通道离开本机**（WS 只回执 questId）。

**回填接口（M4 仅留接口）**：`NoteBackfill.listNotes(filter) / renderNote(questId)`；`injectNote` 不实现（未来候选：`claude --resume` 注入或写入目标项目 `.claude/codestead-notes.md`，前置条件 = 用户逐条确认）。CLI：`codestead notes list / show / open`。

### 9.8 奖励表与降级

| 场景 | 金币 | XP | 道具 |
|---|---:|---:|---|
| AI · decision | **120g** | **60** | — |
| AI · reflection | **80g** | **40** | — |
| 本地题库 | **40g** | **20** | — |
| 首次同意（scripted） | 50g | 20 | — |
| 累计完成 5 / 15 / 30 个 | — | — | 种子礼包×3（`seed_pack_t{1..3}`）/ 装饰「沉思的稻草人」（`deco_thinking_scarecrow`）/ 收藏「村民的信」（`collectible_villagers_letter`）——道具与图鉴形态待 M3 图鉴/装饰子系统定稿后确认，在此之前 M4 按无道具实现 |
| 先不聊 / 不回答 | 0（无任何惩罚） | 0 | — |

互洽性：频率上限 1 任务/15 真实分钟 = 1/5 游戏日 → 金币流 ≤**24g/日 ≈ 日收入 7~12%**（「捎带的谢礼」）；XP 流 ≤12/日 ≈ 农耕 XP 20%；本地题库刻意压到 AI 一半以下；道具均非生产性。奖励常量定义于 `shared/src/quest.ts`，经济子系统同源。

**降级矩阵**：enabled=false → 模块不启动、OFFERED 清场（历史与笔记保留）；claude 不可用 → 静默转本地题库（设置页灰显注明，不弹窗不崩溃）；预算/次数耗尽 → 当日转本地题库次日恢复；连续 3 次失败 → 本地题库模式 + 每 60 分钟探测恢复；WS 断开 → 不生成不积压。**失败处理**：超时 90s SIGTERM+5s SIGKILL；schema 失败不在本次重试（成本纪律），原始输出截断 2KB 写 errors.log；退避 15→30→60 分钟封顶；所有失败对玩家完全不可见（最多「村民今天没找你」）。

**隐私红线**：transcript 读取/脱敏/摘要全在 daemon 进程内；唯一外发 = 脱敏摘要经用户自己的 `claude` CLI → Anthropic API（与日常 Claude Code 同通道）；Codestead 无任何服务器/遥测/第三方端点；WS 载荷只含展示字段、relatedCwd 只推 basename；headless 会话经 `disableAllHooks` + tty 过滤双重隔离，不上 HUD、不自激回环。

验收要点：enabled=false 时 0 次 claude 调用 0 条 quest 消息；任意输入序列相邻生成间隔 ≥cooldown、pending ≤1；脱敏 fixture 0 命中；任务到达瞬间输入焦点/镜头/时间流速零变化；OFFERED 24h 后依然可答；故障注入 ×3 不崩溃自动降级；奖励幂等；首个任务必为 scripted 同意任务。

## 10. 存档与设置

### 10.1 存储方案

> **M1 范围裁决（§0.4）**：M1 = 夜结算必存 + tab 隐藏/blur 去抖（≥5s）存 + 手动存 + JSON 导出/导入（导入 parse/safeParse 失败即拒绝、现有档纹丝不动，不做预览确认屏）；写前 safeParse 自检保留。Web Locks + BroadcastChannel 双标签互斥、backup 轮换 / corrupt 留证 / 三选一恢复对话框、IDB→localStorage→内存降级链、30s 脏轮存、关键事件存档触发器、pagehide best-effort 全部移至 **M5「开源发布」**验收项；迁移链函数数组与 CI 守护测试待 M3 引入 v2 时再建（`schemaVersion` 字段 M1 保留）。

- **存档主存储：IndexedDB**（`idb-keyval`，自建库 `createStore('codestead','kv')`，只用 get/set/del 操作单个版本化 JSON 文档）。理由：容量（worst ≈450KB）、异步不阻塞（visibilitychange 时写档）、单 key put 事务原子；
- **设置：localStorage**（与存档分离）：boot 第一帧前同步可读；删档不重置偏好；损坏静默回退默认值；
- **键位**：IDB `save:slot0`（当前档）/ `save:slot0:backup`（仅日结算轮换，深度 1，**M5**）/ `save:slot0:corrupt`（最近损坏留证，**M5**）；localStorage `codestead.settings.v1`、`codestead.hud.v1`（HUD 全部设置归此，附录 A-21）。slot0 维度为未来多槽预留，M1~M5 单槽；ESLint 把 `idb-keyval` 限制在 `storage/**` 内；
- **降级链（M5）**：IDB → localStorage 适配器（>4MB 拒写提示导出，顶部安静提示「存储受限」）→ 内存模式（可玩但刷新即丢，结算屏高亮导出按钮）；M1 仅 IDB，失败时温和提示导出 JSON。启动时 fire-and-forget `navigator.storage.persist()`，被拒不弹窗。

### 10.4 存档生命周期

**启动加载状态机**：

```
 BOOT（persist()；Web Locks 抢占 = M5）→ LOADING
   ├─ 空 → NEW_GAME（初始值建档立即写盘）→ RUNNING
   └─ 读到 doc：
      ├─ version == CURRENT → VALIDATE(safeParse) ─成功→ RUNNING ─失败→ RECOVERY
      ├─ version <  CURRENT → MIGRATING（副本上逐版迁移，M3 起）─成功→ RUNNING ─失败→ RECOVERY
      └─ version >  CURRENT → TOO_NEW（只读 + 仅导出，绝不写盘/向下迁移）
 RECOVERY（M1 简版）：二选一 [导入 JSON] [开新农场]（现有损坏数据不删除）
 RECOVERY（M5 完整版）：原始数据移入 corrupt 键 → 读 backup →
   可用：三选一对话框 [从备份继续] [导入 JSON] [开新农场]（文案低压，无吓人字眼）
   不可用：二选一 [导入 JSON] [开新农场]
```

**自动存档触发表**：

| # | 触发器 | 节流 | 备注 |
|---|---|---|---|
| A | **日结算存档**（主存档点，NightUpdate #11） | 无（必存） | M1；backup 轮换 M5。写入的是「已过夜」状态——结算屏期间刷新重开直接落在新一天清晨，零状态损失 |
| B | tab 隐藏 / window blur 进入 | 去抖 ≥5s，hidden 立即 | M1；玩家被打断离开的主路径 |
| C | 关键事件（工具升级/扩容/职业/升级/quest 奖励/导入完成） | 5s 合并窗口 | **M5**；不可逆事件不容丢失 |
| D | 脏数据轮存 | 每 30 真实秒检查 dirty | **M5**；「硬关窗口」损失上界压到 30s |
| E | 手动保存 | 立即 | M1 |
| F | pagehide | best-effort | **M5**；B/D 才是真保障 |

**写入原子性**：serialize 不可变快照（异步写绝不暂停 sim）→ 写前 safeParse 自检（失败 = 程序 bug，不写盘保留好档）→ 日结算时先 `backup ← lastPersistedDoc` 再覆盖 slot0（backup 轮换 M5）→ 单在途写 + queued 合并。备份防御的是「逻辑性坏档」，三层恢复 = backup + corrupt 留证 + JSON 导出（前两层 M5；M1 依赖 JSON 导出）。

### 10.5 多标签互斥（M5）

> M5 前同档双 tab 行为未定义（已知限制，文档明示，§2.9）。

Web Locks `'codestead-slot0'`（`ifAvailable`）持锁至页面关闭；没拿到锁 → 「接管」屏（不读写存档）：[在这里继续玩] → BroadcastChannel `'codestead-slot0'` 发 takeover → 旧页立即 save → 停 sim → 释放锁 → 新页等锁（上限 3s，超时提示刷新）。Web Locks 不可用兜底 = BroadcastChannel claim/held 握手（250ms 无回应视为独占）。

### 10.6 版本迁移与导入导出

- `schemaVersion` 从 1 起（字段 M1 即保留）；**任何**字段增删改（含枚举扩值）必升版本，无灰色地带；迁移链 = 纯函数数组（`v → v+1`）存 `shared/src/save.ts`，与 schema 同 PR + fixture 单测；副本上执行、全链成功且终验通过才落盘；迁移前原档写 backup；CI 守护测试断言 1..CURRENT-1 每版迁移函数存在——**迁移链与 CI 守护测试待 M3 引入 v2 时再建**；
- **导出**：SaveDoc 本身 pretty-print（不加包裹/校验和），文件名 `codestead-save-day012-20260610-1430.json`，Blob + `<a download>` 零网络；可编辑视为 feature（不做防作弊）；
- **导入**：M1 = file input → JSON.parse → safeParse，**失败即拒绝、现有档纹丝不动**（不做预览确认屏）；M5 补 migrateSaveDoc + 预览确认屏（双方摘要对比 + 「当前存档将被替换并自动备份」）→ 替换；任一步失败现有存档纹丝不动。

- **五态配色**：取 `shared/src/theme.ts` 单一 token（§7.3，CODE-28 色系）；**双编码纪律：颜色 + 图标形状，任何状态信息绝不只用颜色区分**——不设「色弱模式」开关，默认即安全；验收：灰度滤镜下五态可分，deuteranopia/protanopia 模拟下 blocked 与 done 可分；

- 全部 UI 文本 ≥12 逻辑像素；声音信息均有视觉等价物（muted 不损失信息）；焦点态高对比描边。

### 10.9 存档边界与验收（精选）

QuotaExceeded → 重试 1 次 → 提示导出 + 释放 corrupt 键再试，好档永不被截断覆盖；farmTiles 越界键 → schema regex 拦截进恢复流程；未知 cropId/itemId（版本回退）→ 宽容加载：tile 降级为已开垦无作物、物品移除 + warn；系统时钟回拨 → 无影响（无游戏逻辑读真实时钟）。

验收要点：新档 → 首次日结算 → 强刷 → 落在第 2 天 6:00 状态一致；任意时刻硬关损失 ≤30 真实秒（M5，随脏轮存）、切 tab 损失 = 0；迁移 fixture 全部通过且删任意一环守护测试必红（M3 起，随迁移链）；恢复链三选项可达（M5）；导出→清站点数据→导入完整恢复；删档后设置保持；时钟前调 48h sim 零变化；从新档到 M1 毕业存档/设置系统对话框次数 = 0（恢复流程除外）。

### 11.3 UI 像素风格硬规则

1. UI 与世界共用同一整数 zoom，不做独立高分辨率层；

技术规格（双格式与 loudnorm 为 M3 管线）：双格式 `.ogg`（Vorbis q4）+ `.m4a`（Safari 兜底）；SFX 单文件 ≤100KB / ≤2s；统一响度 ffmpeg loudnorm **SFX −14 LUFS** / true peak −1.5dBTP；**防机枪**：同 key 50ms 去重（M1 即有）+ 音高抖动 ±4% + 并发上限 3（M3）。

### 11.6 BGM 方案（M3；M1 无 BGM）

> AudioDirector 五态状态机、BGM 三首与奇偶轮换、连击半音爬升（§6.4）整体移 M3。

3 首 + 静音：`bgm/day_a` / `day_b`（90~150s 循环，70~100 BPM 大调民谣质感、无鼓点驱动，按游戏日奇偶轮换防洗脑）/ `rain_day`（60~80 BPM 柔和，与 rain_loop 叠放）。OGG q3 + m4a；**−16 LUFS**；单轨 ≤3.5MB；**首屏不加载 BGM**（首次手势解锁后惰性加载）。

**AudioDirector 状态机**：

```
[SILENT] ─首次手势解锁&加载完─► [DAY_CALM(A|B 奇偶)] ◄─WeatherChanged─► [RAIN_DAY]
    ▲                                │     （交叉淡出 800ms / 淡入 600ms）
    │ 设置关闭音乐                DayEnded ▼
    └──────────────── [SUMMARY]：BGM 淡出 800ms → jingle_day_end 一次 → 静音
                          │ DayStarted（关结算屏）→ 回 DAY_CALM（淡入 600ms）
 任意态 ─SimPaused(tab 隐藏/菜单)─► [SUSPENDED]（AudioContext.suspend()）─SimResumed─► 恢复原状态（不重播）
```

雨环境声独立总线：雨起淡入 1.5s / 雨停淡出 1.5s。**音频解锁**：LOCKED（不发 play 不报错）→ 首次输入 → resume + 惰性加载；解锁前的音效直接丢弃不排队；resume 失败 → 全局静音降级。总线默认见 §10.7；音量分层：世界动作 1.0 / UI 0.8 / jingle 1.0 / blip 0.25。

```

```

体积预算（CI）：atlases ≤1.5MB｜tileset ≤256KB｜字体 ≤4MB｜SFX+jingle ≤2.5MB｜BGM+ambience ≤12MB（音频体积预算与双格式 CI 检查随 M3/M5 音频管线启用）｜**assets 总计 ≤20MB，首批加载 ≤8MB**；冷启动可操作 ≤3s。

管线：**M1 = Aseprite 手工导出单图集 + 手维护 manifest.json**（逐文件许可追溯红线不变，自动化校验后补）；`pnpm assets:import`（sharp + maxrects-packer + tile-extruder，读 recipes.json5）与 sha256 重现校验**整体移 M5**；`assets:audio`（loudnorm + 双格式）随 M3。`check:assets`（CI 门禁：① 每文件有 manifest 条目；② license ∈ 白名单且 redistributable；③ recipes 重切 sha256 一致（M5）；④ 体积预算；⑤ 帧名正则）。manifest 条目含 author/source/sourceSha256/license/licenseVerifiedAt/derivedFrom/processing；`ATTRIBUTION.md` 按来源逐项列出（FreePD 逐曲、freesound 逐条），M5 根 CREDITS 链接。
