## 2. 物品分类体系

### 2.1 类别总表

| category | 中文 | 堆叠上限 | 可出售 | 可丢弃 | 进入里程碑 | 说明 |
|---|---|---:|---|---|---|---|
| `tool` | 工具 | 1 | 否 | 否 | M1 | 锄头、喷壶；档位（tier）存在工具状态表，不是独立物品（§2.3） |
| `seed` | 种子 | 99 | 是（购价 ×0.5 向下取整） | 是 | M1 | 选中后 E/点击 = 播种 |

### 2.3 数据结构（`packages/shared` / `game/src/sim/items.ts`，纯 TS 零 Phaser）

```ts
type ItemCategory = 'tool' | 'seed' | 'crop' | 'material' | 'quest';

interface ItemDef {
  id: string;                 // "turnip_seeds"
  nameKey: string;            // i18n key
  category: ItemCategory;
  stackMax: 1 | 99;
  sellPrice?: number;         // undefined = 不可出售
  discardable: boolean;       // tool/quest = false
  iconFrame: string;          // 图集帧名（manifest.json 可追溯）
  cropId?: string;            // seed → 对应 CropDef.id
}

interface ItemStack { itemId: string; count: number }   // 1 <= count <= stackMax

interface InventoryState {
  slots: (ItemStack | null)[];   // 长度 = capacity，索引 0..8 即快捷栏
  capacity: 12 | 24;             // M1 固定 12；M3 扩容 24（1,000g，宪法 §4.4）
  selected: number;              // 0..8，当前选中的快捷栏格
}

interface ToolTiers { hoe: 1 | 2 | 3; wateringCan: 1 | 2 | 3 }  // 1=木 2=铜 3=金
```

工具升级（500g→铜、2,500g→金，即时生效）只改 `ToolTiers`，背包里的 `hoe` 物品不变、图标按档位换帧——避免「升级瞬间物品 id 变更」带来的存档与引用问题。

## 3. 背包与快捷栏

### 3.1 容量模型

- **背包总容量 12 格（M1）**；M3 花 1,000g 扩容至 24 格（一次性，即时生效）。
- **快捷栏 = 背包前 9 格**（索引 0–8），常驻屏幕底部，数字键 1–9 与滚轮选中（宪法 §4.5）。
- 索引 9–11（M1）/ 9–23（M3）为**储备格**，仅在背包界面可见，通过拖拽/快速移动与快捷栏交换。
- 背包界面网格固定 9 列：第 1 行 = 快捷栏（带 1–9 角标），第 2 行起 = 储备格；M1 第 2 行只有 3 个可用格，其余 6 格画成「锁定格」（暗色 + 锁图标，悬停提示「农场升级后可扩容」）——把 M3 的扩容做成可见的明日之诺。

### 3.2 选中规则

- 任意时刻**恰好有一个**快捷栏格处于选中态（含空格——空格 = 空手）；新档默认选中 1 号位（锄头）。
- 切换方式：数字键 1–9 直选；滚轮向下 = 右移、向上 = 左移，1↔9 循环；切换在下一渲染帧生效（≤1 帧）。
- 切换反馈：高亮框移动 + 选中格内容上浮 2px + 短「嗒」声；快捷栏上方浮现物品名 1 秒（8px 字）。
- 选中物决定世界交互语义（§5.3 优先级表）；选中状态入存档（§9）。

### 3.3 堆叠与合并

- 拾取/购买入包顺序：**先填同 id 未满堆**（快捷栏从左到右 → 储备格从左到右），再占**第一个空格**；一次入包可拆为多堆。
- 部分入包：能放多少放多少，返回 `{added, rejected}`；rejected > 0 时由调用方决定行为（收割 = 整笔阻止，见 §8.1；购买 = 降级数量，见 §6.6）。
- 整理（背包界面 R 键 / 「整理」按钮）：合并所有可合并堆 → 按 `tool < seed < crop < material < quest`、同类按 itemId 字典序排列；**只整理储备格，不动快捷栏**（玩家手排的工具位是肌肉记忆，不可破坏）。

### 3.4 sim 层接口（`game/src/sim/inventory.ts`）

```ts
interface InventoryApi {
  add(itemId: string, count: number): { added: number; rejected: number };
  canAdd(itemId: string, count: number): boolean;   // 收割/购买前置检查
  removeAt(slot: number, count: number): boolean;
  move(from: number, to: number): void;             // 含交换/合并语义
  select(hotbarIndex: number): void;                // 0..8
  sort(): void;
  // 事件（Phaser.Events.EventEmitter，渲染层订阅）：
  // 'inventory:changed' {slots}  'inventory:full' {itemId}
  // 'hotbar:selected' {index, itemId|null}
}
```

确定性要求：所有方法纯逻辑、无随机、无真实时钟，可在 Node 中 headless 测试（tech-stack §1 测试策略）。

## 4. 丢弃：垃圾桶 + 单步反悔

- **不做世界掉落物**：丢弃 = 销毁，不在地图上生成可拾取实体（避免「地上有东西要捡」的隐性待办，违反低压定位；也省一套实体系统）。
- 丢弃唯一入口：背包界面右下角**垃圾桶格**。手持物品点击垃圾桶 → 整堆销毁；`discardable: false`（工具/任务道具）→ 拒绝 + 摇头动画 + 提示「工具不能丢弃」。
- **撤销槽（零焦虑设计）**：垃圾桶旁有一个「↩」撤销格，显示**最近一次**被销毁的堆；点击即取回（按入包规则放回，放不下则回到光标手持态）。撤销槽在以下时机清空：再次丢弃（被新内容顶替）、关闭背包界面、日结算。撤销槽不入存档。
- 无丢弃确认弹窗：撤销槽已经提供反悔路径，确认弹窗只会增加日常摩擦。

## 5. 拾取与使用手感

### 5.1 获得物品管线（无地面掉落）

收割/购买/任务奖励一律**直接入包**。收割时序（总时长约 0.5s，不阻塞移动）：

| t (ms) | 表现 |
|---:|---|
| 0 | 作物精灵上弹 12px、缩放 1→1.1→0，260ms；锄/收音效（±10% 随机音高） |
| 60 | 物品 icon 从作物格沿贝塞尔弧飞向目标快捷栏格，240ms |
| 300 | 槽位 bounce（scale 1→1.25→1，120ms）+ 计数刷新 + 「+1 芜菁」浮字（上飘 12px，600ms 淡出） |
| 300 | 「pop」入包音；**连击音阶**：1s 内连续入包每次 +1 半音，封顶 +7（连收一排地的爽感核心） |

XP 获得（宪法 §4.4 公式）同帧发放，浮字与「+1」并排（「+8 XP」，颜色区分）。

### 5.2 工具使用手感

- **动作时长 250ms**：0ms 起手 → 120ms 效果落地（地块变化、音效、粒子）→ 250ms 收尾。动作期间锁移动。
- **输入缓冲 150ms**：动作未结束时按下的下一个动作/移动入队 1 个，结束即执行——连续操作无「按了没反应」。
- **按住连发**：按住 E / 按住左键，每 280ms 对当前面朝格/光标格重复动作（浇一排地不用搓键）。
- 粒子与音效：锄地 = 4~6 个土块粒子 + 闷响；浇水 = 水弧 + 地块湿色 tint + 水声；播种 = 3 粒种子粒子 + 立即出现 stage0 苗（「种下去了」必须当帧可见）。
- 范围预览：铜/金档工具选中时，面朝方向常显半透明范围框（直线 3 格 / 3×3），E 按下前就知道会浇到哪。

### 5.3 目标格与交互优先级

- **面朝格高亮框**常驻：绿框 = 当前选中物可作用；灰框 = 不可作用。判定走 sim 查询 `canActOn(tile, selectedItemId)`。
- 鼠标交互范围 = 以玩家为中心 **3×3（切比雪夫距离 ≤1）**；点击范围内格子先把角色转向该格再执行（同帧）；**超出范围不寻路**，目标格闪红框 0.3s（防误点乱跑，星露谷同款取舍）。
- E 与左键**全程等价、可混用**（宪法 §4.5）：E 作用于面朝格，左键作用于光标格（限 3×3）。
- 同一格多语义时的优先级（高 → 低）：

| 优先级 | 条件 | 动作 |
|---|---|---|
| 1 | 格上有交互实体（商店柜台/床/NPC M4） | 对话/打开界面 |
| 2 | 格上有成熟作物 | **收割（无论手持何物）**——防止手持锄头误铲熟地 |
| 3 | 手持种子 + 格为已锄空地 | 播种 |
| 4 | 手持工具 + 工具可作用 | 锄地/浇水（已湿格浇水 = 无操作 + 轻提示音） |
| 5 | 其余 | 无操作（灰框，无声） |

## 6. 通用 UI 框架

### 6.1 UI 栈状态机

单一模态栈 `uiStack`，**不变量：`uiStack.length > 0 ⇔ sim 暂停（原因 "ui"）`**；`document.hidden` 独立贡献暂停原因 `"hidden"`（引用计数，宪法 §4.3 时间即停）。

```
栈顶状态        输入/事件                  动作
─────────────────────────────────────────────────────────────
(空)=游玩      Esc                       push 暂停菜单
(空)           Tab 或 I                  push 背包
(空)           E/点击 商店柜台            push 商店
(空)           22:00（时间子系统触发）     push 日结算屏（唯一系统自动弹出）
暂停菜单        Esc / 「继续」             pop
暂停菜单        「设置」                   push 设置
暂停菜单        「回到主菜单」             自动保存 → 清栈 → 主菜单场景
背包           Esc / Tab / I             手持物归位 → pop
商店           Esc / 「离开」             清回购栏 → pop
设置           Esc / 「返回」             pop（改动即时生效并持久化）
日结算屏        任意键/点击（开屏 400ms 后） pop → 新的一天 6:00
─────────────────────────────────────────────────────────────
规则：模态期间世界输入（移动/E/1-9/滚轮选格）全部屏蔽；
     滚轮在列表界面转为滚动列表；日结算屏的 400ms 输入宽限防连按误关，
     之后停留不限时（宪法：无倒计时）。
```

### 6.9 拖拽（光标手持）状态机

```
状态: EMPTY（空手） / HOLD(stack)（图标跟随光标，offset +4,+4）

EMPTY:
  左键@有物格   → 拿起整堆 → HOLD
  右键@有物格   → 拿起 ⌈n/2⌉，原格留 ⌊n/2⌋ → HOLD
HOLD(k):
  左键@空格     → 放下整堆 → EMPTY
  左键@同id格   → 合并至 99 上限，溢出留手（k>0 维持 HOLD）
  左键@异id格   → 交换（手持 ↔ 格内）
  右键@空格/同id格 → 放下 1 个（k-1；k=0 → EMPTY）
  Shift+左键@格 → 快速移动：快捷栏↔储备区第一个可用位（商店卖出tab=卖整堆）
  点击垃圾桶    → discardable? 销毁→撤销槽 : 拒绝动画（→ 维持 HOLD）
  Esc/关闭界面  → 归还原格（占用则第一空格）→ EMPTY
```

### 6.10 toast 与反馈规范

- 位置：快捷栏上方居中（y=312）；最多同屏 2 条，超出排队；展示 1.2s + 0.3s 淡出；1s 内重复文案去重。
- toast 只用于「动作被阻止的原因」（背包已满/金币不足/工具不能丢弃）；成功反馈一律用就地动效（浮字/bounce/音效），不发 toast——成功不打扰。
- 全局禁止任何确认弹窗，例外仅两处：新游戏覆盖存档、删除存档（双重确认）。

## 10. 验收标准（出厂自检，对照宪法 §7 红线）

1. **双通道完成核心循环**：分别用「纯键盘（零鼠标）」与「键盘移动 + 纯鼠标交互」各完成一次 锄→种→浇→收→卖，全程无任何一步只能用另一通道完成；
2. 1–9 直选、滚轮循环、点击快捷栏三种切换方式均 ≤1 渲染帧生效，且高亮/上浮/音效齐全；
3. 收割连发 9 格：连击音阶爬升、入包动画不丢帧、计数准确；第 10 格在背包满时被阻止且作物零损失，关页重开后该作物仍可收（红线 #3）；
4. 打开背包/商店/菜单/设置任意界面：右上角时钟停走；关闭后从停点继续；`document.hidden` 同样停走（红线 #3）;
5. 拖拽 fuzz 测试（随机 1,000 次拿起/放下/交换/Esc 关闭/丢弃/撤销序列后）：背包内物品总数守恒（仅垃圾桶销毁路径减少，且每次销毁先经撤销槽）；
6. 丢工具、卖工具、卖任务道具三条路径全部被拦截并有反馈；
7. 商店：金币不足/背包满时购买正确降级或阻止，无负数金币、无超堆叠；误卖后经回购栏原价取回零损失；
8. 整理只动储备格、快捷栏顺序不变；
9. 存档：手动保存→刷新页面→载入，背包内容、选中格、工具档位、容量完全一致；构造坏档（未知 itemId、count=999）载入不崩溃且自动修复；
10. 全程无一次自动弹出的模态（日结算除外）、无成功类 toast、左上角预留区无任何本子系统像素（红线 #4）；
11. UI 文本全部走 i18n key，位图字体覆盖 M1 全部用字。

## 11. 开放问题

1. **新档是否赠送 5 包小萝卜种子**（slot 3）？赠送可省去新手第一趟商店往返，更稳地达成「10 分钟红线」；但买种子本身是商店教学。建议由 farming/新手引导稿裁决，本文倾向赠送。
2. **出售渠道单一化**：本文假设唯一出售路径 = 商店卖出 tab，不做出货箱（shipping bin）。若 farming/地图稿希望院子里有出货箱（少跑路），需补「出货箱 = 卖出 tab 的简化版」界面，金币入账时机（即时 vs 日结算）需与时间稿对齐。
3. **喷壶水量**：M1 假设无限水（mechanics §9.4 未列容量）。若 farming 稿引入水量/补水，快捷栏喷壶格需加水量条 UI，本文预留槽位下沿 2px 进度条规范。
4. **改键（键位自定义）**：M1 固定键位，是否列入 M5？涉及设置界面扩展与存储，建议 M5 评估时一并裁决。
5. **CJK 位图字体管线**：BMFont 子集生成（按 i18n 文案扫描出字表）需要一个构建脚本，归属 game 包构建链；M5 英文版需第二字表。需要技术稿确认工具选型（如 msdf-bmfont 之外的像素友好方案）。
