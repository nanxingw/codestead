# 10 — 世界与地图（world-map）设计稿

> 版本 v1.0 ｜ 2026-06-10 ｜ 状态：草案
> 负责范围：农场地图布局、村庄/集市区域、tilemap 图层规划、相机行为、角色移动、面向格子的交互模型、新手前 10 分钟空间动线。
> 上位约束：`00-constitution.md`（16×16 正交、640×360 整数缩放、四向移动、E/鼠标等价、M1 单图 ≤64×64、零焦虑承诺）。冲突时以宪法为准。

---

## 1. 设计目标（对照三支柱）

1. **游戏第一**：地图本身要有「探索一眼可见、动线短、处处是明日之诺」的空间叙事——锁着的田、待建的空地、修不完的南门，都是看得见的下一步；
2. **零焦虑**：地图上没有任何会随真实时间变化的东西；所有解锁只由农场等级驱动，解锁只**移除**碰撞、永不添加（玩家不可能被新碰撞卡住）；
3. **感知不干预**：地图与 HUD/quest 零耦合；仅为 M4 预留 NPC 锚点对象层，M1~M3 不渲染任何 NPC。

## 3. Tilemap 图层规划

### 3.1 图层表（farm.tmj）

| # | 图层名 | 类型 | 渲染深度 (depth) | 内容 | 碰撞 |
|---|---|---|---|---|---|
| 1 | `ground` | tile | 0 | 草地、泥土路、水面、田区底色 | 无 |
| 2 | `farmland` | **运行时生成**（`map.createBlankLayer`，不在 .tmj 内） | 10 | 已锄地、已浇水的覆盖 tile（由 farming 子系统驱动） | 无 |
| 3 | `ground-detail` | tile | 20 | 草丛、花、路缘、石子等贴地装饰 | 无 |
| 4 | `buildings` | tile | 30 | 农舍墙体下半、水井、摊位柜台、栅栏、出货箱底座 | 经 `collision` 层表达 |
| 5 | `entities` | Phaser Container（非 Tiled 层） | **100 + 脚底 worldY**（y-sort） | 玩家、作物精灵、未来 NPC/动物 | 玩家 AABB |
| 6 | `above` | tile | 1000 | 屋顶、树冠、摊位遮阳棚——玩家可从其后走过 | 无 |
| 7 | `fx` | Phaser 层（非 Tiled） | 1100 | 雨滴粒子、昼夜色调叠层（时间/天气子系统驱动） | 无 |
| 8 | `collision` | tile（**不可见**） | 不渲染 | 单一保留 gid 标记不可走格；加载后 `setCollisionByExclusion([-1])` | **唯一碰撞事实源** |

对象层（不渲染，构建期提取）：

| 对象层 | 内容 |
|---|---|
| `spawn` | `player_spawn` 点 (27,11)，属性 `facing: "down"` |
| `zones` | 矩形：`field_a`、`field_b`(unlockLevel=3)、`field_c`(unlockLevel=5)、`market`、`build_coop`、`build_workshop`、`build_greenhouse`、`village_exit` |
| `interactables` | 点/矩形：`house_door`、`shipping_bin`、`well`、`shop_stall`、`bulletin_board`、`signpost_junction`、`gate_sign`，各带 `kind` 属性 |
| `water-sources` | 池塘岸边与水井邻格，属性 `waterSource: true` |
| `npc-anchors` | M4 预留：`market_stall`、`bulletin`、`pond_bench`、`field_edge` 四个站位点 |

### 3.2 深度（z-order）纪律

- 所有「会被玩家绕到身后」的物体（作物、未来 NPC、建筑前景件）一律进 `entities` 容器，`depth = 100 + 精灵脚底 worldY`，每帧对移动中的精灵更新（静态作物只在种植/生长时更新一次）；
- 屋顶/树冠只进 `above`，绝不进 `entities`——避免 y-sort 与大型静态物混算；
- `farmland` 运行时层只放「锄地 / 已浇水」两类 tile（湿土用深色变体 tile，**不用 alpha 叠加**，保证像素风干净）。

### 3.3 地图 → sim 的构建期契约（保持 sim 零 Phaser）

新增脚本 `packages/game/scripts/export-map-meta.ts`（pnpm build 前置步骤）：读取 `farm.tmj`，产出 `packages/game/src/sim/data/farm-map-meta.json`：

```ts
interface MapMeta {
  width: 64; height: 48;
  tillable: Rect[];                       // 田 A/B/C 矩形
  unlockGroups: { zoneId: string; farmLevel: number; rects: Rect[] }[];
  waterSources: TilePos[];
  spawn: { tile: TilePos; facing: Facing };
  interactables: { id: string; kind: string; tiles: TilePos[] }[];
  buildPlots: { id: string; rect: Rect }[]; // M3
  npcAnchors: { id: string; tile: TilePos }[]; // M4
}
```

- sim 层只 import 这份 JSON，不 import phaser、不解析 .tmj（ESLint `no-restricted-imports` 纪律不破）；
- `collision` 层**不**导出给 sim：碰撞是渲染/输入侧关注点，sim 只关心格子的农业语义；
- CI 校验脚本（tech-stack 风险 #9）同时检查：正交、非 infinite、tileset 嵌入、CSV 编码、图层名与本表一致、`zones` 与 `tillable` 矩形不与 `collision` 冲突。

## 4. 角色移动

### 4.1 数值表

| 项 | 数值 | 说明 |
|---|---|---|
| 移动模型 | **自由像素移动**（Arcade Physics AABB），非格子锁定 | 操作手感星露谷式；农业交互仍按格子结算 |
| 方向 | **四向**（宪法 §4.5），无对角移动 | 同时按住两轴时**后按下的键生效**（输入栈，见 §4.2） |
| 步行速度 | **72 px/s = 4.5 tiles/s** | 横穿全图（64 tiles）≈ 14 s |
| 奔跑速度 | **120 px/s = 7.5 tiles/s**，按住 Shift | 宪法未列但不冲突（纯增益）；可在设置中关闭，见开放问题 |
| 精灵尺寸 | 16 × 16 px，锚点 (0.5, 1)（脚底中心） | Kenney 角色；行走动画不足时用 2 帧上下 1px 颠簸代替 |
| 碰撞盒 | **12 × 8 px**，对齐脚底：offset (2, 8) | 窄于 16px tile → 1 格宽缺口（门、栅栏口）可顺畅通过 |
| 贴角宽容 | 撞角重叠 ≤ 3 px 时沿垂直轴每帧推 1 px 滑过 | 消除「卡在栅栏角」挫败感 |
| 当前格 | `floor(碰撞盒中心 / 16)` | 朝向/交互/存档均以此为准 |
| 存档字段 | `player: { x: number; y: number; facing: Facing }`（像素坐标） | 进存档 schema（依赖 storage/save 设计） |

### 4.2 输入栈（四向冲突消解）

维护按下顺序栈 `heldDirs: Facing[]`：按下入栈、松开出栈；**栈顶**为当前移动方向；同帧多键按固定序 `up > down > left > right` 入栈（确定性，便于回放测试）。朝向 `facing` 在移动方向变化时同步更新；静止时保持最后朝向。

### 4.3 玩家状态机

```
            按住方向键                松开方向键
   ┌────────┐ ─────────────► ┌────────┐
   │  Idle  │ ◄───────────── │  Walk  │
   └───┬────┘                └───┬────┘
       │  E / 左键（目标格动作有效）   │ 同左
       ▼                          ▼
   ┌──────────────────────────────────┐  动作完成（250ms）
   │ Acting：锁定移动，播动画+音效        │ ────► 回 Idle / Walk（方向键仍按住则直接 Walk）
   │ 期间方向键状态保留，新动作输入丢弃     │
   └──────────────────────────────────┘
   任意状态 ──Esc/对话/日结算打开──► Suspended（模拟 tick 全停，宪法 §4.3）──关闭──► 原状态
   任意状态 ──22:00──► Sleeping（400ms 渐隐 → 日结算屏）──新一天 6:00──► Idle @ (27,11) 面朝下
```

- `Acting` 锁 **250 ms**（锄地/浇水/播种），徒手收割 **200 ms**；
- **长按连发**：按住 E 或左键 ≥ 400 ms 后，每 **300 ms** 自动对当前目标格重复一次动作（目标无效则跳过不计时惩罚）——这是「2 分钟浇完 12 格水」验收的关键 QoL；
- 动作队列深度 1：锁定期内最后一次点击不缓存（防连点漂移）。

## 5. 面向格子的交互模型

### 5.1 目标格判定

```
function resolveTargetTile(player, aimMode, hoverTile):
  base = player.currentTile
  if aimMode == KEYBOARD:
    t = base + DELTA[player.facing]          // 面朝格
    if !sim.queryAction(t, activeItem).valid
       && sim.queryAction(base, activeItem).valid:
      t = base                               // 脚下格回退（站在成熟作物上按 E 可收割）
  else: // MOUSE
    t = (chebyshev(hoverTile, base) <= 1) ? hoverTile : null   // 3×3 触及范围
  return t
```

- **键鼠等价定义**：两种输入最终都收敛为同一条 `interact(tile, activeItemId)` 命令派发给 sim；E = 面朝格（含脚下回退），鼠标 = 身边 3×3 任意格。鼠标可达集合是键盘的超集，属可接受的不对称（星露谷同型）；
- 点击 3×3 内的格子会同时把 `facing` 转向该格（不移动）；
- **点击 3×3 之外：不寻路、不移动**，目标格闪一次灰色边框（120 ms）示意「太远」——M1 不做自动寻路（开放问题 #5）。

### 5.2 瞄准模式状态机

```
┌──────────────┐  鼠标移动 > 8px 或 任意鼠标键   ┌───────────┐
│ KEYBOARD 瞄准 │ ─────────────────────────► │ MOUSE 瞄准 │
│ 光标=面朝格    │ ◄───────────────────────── │ 光标=悬停格 │
└──────────────┘  WASD/方向键/E 任意按下        └───────────┘
```

最后使用的设备获胜；切换瞬间光标平滑跳到新目标格（无补间，像素风直接换位）。

### 5.3 格子光标（高亮）视觉规格

16×16 描边框（1px 内描边，UI tileset 中的 9-slice 帧），渲染于 depth 90（`farmland` 之上、`entities` 之下）：

| 状态 | 视觉 | 含义 |
|---|---|---|
| 有效 | 白框 + 内部 12% 白色填充，0.8s 呼吸（alpha 0.7↔1.0） | 当前工具/物品对该格有动作（`sim.queryAction().valid`） |
| 无动作 | 灰框 alpha 0.4，无填充 | 瞄着但当前工具无事可做 |
| 太远（仅鼠标） | 灰框单次闪烁 120 ms | 点击了 3×3 之外 |
| 隐藏 | 不渲染 | 菜单/对话/日结算打开时；Sleeping 时 |

光标旁不挂任何文字提示（低认知负荷）；动作动词反馈交给动作音效与动画本身。

```
 键盘瞄准（面朝下）：                鼠标瞄准（3×3 触及范围示意）：
      ┌──┐                            . . . . .
      │P │  P=玩家                     . ┌─────┐ .   框内任意格可点
      └──┘                            . │. P .│ .
      ╔══╗ ← 白色光标=面朝格            . │. .▣│ .   ▣=悬停格(白框)
      ╚══╝                            . └─────┘ .
```

### 5.4 与 sim 的接口（假设，待 farming 稿对齐）

```ts
// game 渲染侧 → sim（命令）
sim.dispatch({ type: 'interact', tile: TilePos, itemId: string });
// sim → 渲染侧（查询，纯函数，光标每帧调用）
sim.queryAction(tile: TilePos, itemId: string): { valid: boolean; verb: 'till'|'sow'|'water'|'harvest'|'sell'|'talk'|'none' };
// sim → 渲染侧（事件，驱动 farmland 层与作物精灵）
on('tileChanged', (tile, tileState) => ...);   // tilled/watered/crop stage
on('zoneUnlocked', (zoneId) => ...);           // 移除栅栏 + 粒子
```

固定交互物（出货箱/摊位/门/告示牌/水井）由 map 侧据 `interactables` 对象层注册 `kind`，`interact` 命中时直接发对应 UI 事件（开商店、开出售箱、睡觉确认、看公告），不经过农业 sim。

## 6. 相机行为

| 项 | 规格 |
|---|---|
| 跟随 | `camera.startFollow(player, true, 0.12, 0.12)`——平滑 lerp，无死区 |
| 取整 | `roundPixels: true`（全局），相机滚动值最终取整，杜绝纹理裂缝与抖动 |
| 边界 | `setBounds(0, 0, 1024, 768)`；地图两轴均大于视口，永远贴边裁剪，无黑边露出 |
| 缩放 | 相机 zoom 恒为 1；整数倍缩放全部交给 Scale Manager（宪法 §4.2：FIT + 最大整数 zoom + letterbox） |
| 日出/入睡 | 6:00 `fadeIn(400ms, 黑)`；22:00 入睡 `fadeOut(400ms)` → 日结算屏 |
| 解锁演出 | 田 B/C 解锁的清晨**不**移动相机去看栅栏（不干预）；仅当栅栏在视口内时播放粒子 |
| 禁止项 | 无相机震屏、无缩放动画、无强制运镜——「可冥想」基调 |
| 窗口 resize | 重算整数 zoom，相机保持玩家居中；模拟不暂停（resize 不是离开） |
| 昼夜色调 | `fx` 层全屏叠色（时间子系统驱动）：06:00 暖橙 8% → 10:00 无 → 17:00 暖黄 6% → 20:00 暮蓝 15% → 22:00 暮蓝 25%；线性插值，纯视觉不影响判定 |

## 7. 新手前 10 分钟空间动线（≈3.3 游戏日）

设计原则：**第一屏即全教学**——出生点 (27,11) 的初始视口（40×22.5）同时看见：农舍、出货箱、起始田 A、道路与路口指路牌，集市方向有视觉引导（道路 + 牌子「→ 集市」）。

关键距离（步行 4.5 tiles/s）：

| 路线 | 距离 | 耗时 |
|---|---|---:|

## 9. 边界情况

1. **解锁瞬间玩家贴着栅栏**：解锁只移除碰撞 tile，不可能把玩家挤进碰撞体；无需位置校正；
2. **tab 切走再回**：模拟 tick 停（宪法）；渲染 delta 钳制 ≤ 100 ms，玩家不会瞬移；`Acting` 计时一并冻结；
3. **菜单/对话打开时点击地图**：输入被 Suspended 状态吞掉，光标隐藏；
4. **站在作物格上收割**：键盘走「脚下格回退」（§5.1）；鼠标直接点脚下格；收割后该格仍可走（无新增碰撞）；
5. **长按连发扫过无效格**：跳过无效目标不重置 300 ms 节拍，连续浇一排水不打嗝；
6. **同帧按下相反方向键**：输入栈固定序消解（§4.2），行为确定可测试；
7. **窗口极小（< 640×360）**：zoom 取 1，FIT letterbox 缩小画布——可玩性下降但不崩溃；README 标注最低窗口；
8. **存档位置非法**（地图改版后落在碰撞内）：加载时检测 `collision` 命中 → 重置到出生点；
9. **22:00 正在 Acting**：当前动作完成后再进入 Sleeping（最长延迟 250 ms，不打断动作反馈）；
10. **村庄南门**：交互显示施工牌对话（暂停时间），文案保持温暖（「路还在修，不急。」），明确这是未来内容而非 bug。

## 10. 验收标准（world-map 出厂自检）

1. `farm.tmj` 通过 CI 校验脚本：64×48、正交、非 infinite、tileset 嵌入、CSV、图层/对象层名与 §3.1 完全一致；

3. 出生视口内同时可见：农舍、出货箱、田 A、指路牌（截图验收）；
4. 实测步行：出生点→集市 ≤ 8 s；田 A→出货箱 ≤ 3 s；
5. 四向移动无对角位移；12px 碰撞盒可通过任意 1 格宽缺口；贴角不卡死；
6. E 与鼠标点击在 3×3 内派发**同一条** interact 命令（单测：两种输入产生相同 sim 命令序列）；
7. 光标在键/鼠模式切换、工具切换、菜单开合下始终准确（含脚下回退与「太远」反馈）；

9. 田 B/C 解锁无需重载地图，解锁前后均无任何新增碰撞把玩家困住（自动化测试：解锁前后 flood-fill 可达区域只增不减）；
10. tab 隐藏 / Esc 菜单 / 日结算期间：玩家位置、Acting 计时、相机全部冻结，恢复无跳变；
11. 全程无相机震屏/强制运镜/弹窗（对照宪法红线 #4）；
12. 60 FPS @ 整数 zoom ×2/×3（M1 体量下为冒烟项）。
