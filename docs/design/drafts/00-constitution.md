# Codestead 设计宪法

> 版本 v1.0 ｜ 2026-06-10 ｜ 状态：草案
> 本文件是后续所有子系统设计的**最高约束**：任何子系统设计与本文冲突时，以本文为准；要突破先修宪（PR 中说明理由与代价）。
> 依据：CLAUDE.md、`docs/design/research/mechanics.md`、`docs/design/research/assets.md`、`docs/design/tech-stack.md`。

## 1. 核心幻想与基调（一句话）

**「等 AI 干活的间隙，回到一块永远不催你的小农场」**——星露谷风格的低压像素田园：温暖、连贯、可冥想，时间只为你流动。

## 2. 玩家画像

等 AI 干活的程序员，在桌面浏览器标签页里游玩：

| 维度 | 规格 |
|---|---|
| 单次游玩时长 | 2~10 分钟（= 0.7~3.3 个游戏日，必跨至少一次「过夜」回报） |
| 中断方式 | 随时被打断（agent 转 blocked/done 即离开），离开必须零损失 |
| 回访频率 | 一天多次（典型 5~15 次），每次回访必有「收获 + 明日之诺」 |
| 设备与姿势 | 桌面浏览器 + 键鼠，与工作终端同屏或相邻屏 |
| 心理诉求 | 替代刷手机/刷推的碎片信息流；要低认知负荷、零 FOMO、不添加第二重焦虑 |

## 3. 设计三支柱

所有子系统设计必须逐条对照三支柱，违反任一即打回：

1. **游戏自身成立（游戏第一）**：玩法经得起脱离「监控工具」语境的独立评价；任何一次 2 分钟游玩至少推进一条线（金币/XP/图鉴/建造），并留下一个明日之诺。
2. **零焦虑承诺（随时放下）**：现实时间永不驱动任何游戏状态——无枯萎、无限时、无离线惩罚（FarmVille 教训，mechanics.md §8.2）；日结算屏是设计出来的「放下点」；关页/切 tab/开菜单即暂停且零损失。
3. **感知不干预**：会话状态只展示、永不打断——HUD 不弹窗、不强制暂停、不绑定游戏经济；AI 关卡频率克制、可整体关闭；工作内容（transcript、关卡上下文）永不出本机。

## 4. 全局硬规格

### 4.1 tile 尺寸与视角
- **顶视角正交**（orthogonal top-down）；tile **16×16 px**；角色精灵 16×16 体系；
- 地图管线：Tiled 1.12.2 + `.tmj`（tileset 嵌入、图层 CSV 不压缩、禁 infinite、仅正交）；M1 农场地图 ≤ **64×64 tiles**。

### 4.2 目标分辨率与像素缩放
- 逻辑分辨率 **640×360**（16:9，可视约 40×22.5 tiles）；
- **仅整数倍缩放**：×2=1280×720、×3=1920×1080；Phaser `Scale.FIT` 下取不超过窗口的最大整数 zoom，余量 letterbox 补黑边；
- `pixelArt: true`（最近邻采样）+ `roundPixels: true`；全游戏禁用抗锯齿与非整数像素位移。

### 4.3 游戏内时间与「时间即停」铁律
- 游戏日 **6:00–22:00**（16 游戏小时）；**1 游戏分钟 = 0.1875 真实秒**（10 游戏分钟 = 1.875 秒），**1 游戏日 = 3 真实分钟**；一季 28 游戏日（M1 仅春季，换季规则 M3 后启用）；
- 22:00 自动入睡（无晕倒、无任何惩罚）→ **日结算屏**：今日收成/收入/明日预告，并平静展示会话状态；停留不限时、无倒计时；存档随日结算自动落盘（IndexedDB）+ 随时手动存；
- **关掉页面时间即停**：tab 隐藏（visibilitychange）、菜单/对话/结算屏打开 → 模拟 tick 全停；禁止任何由真实时钟驱动的游戏机制（作物生长、建造工期一律按游戏内时间计）。

### 4.4 经济基准
- 初始金币 **100g**；作物定价公式 `goldPerDay = (sellPrice × expectedYield − seedPrice / harvestCount) / totalDays`；
- 基准 **T1 = 4.0 g/格·日**，每升一档 +50%~70%（T2 ≈ 6.5~7.5，T3 ≈ 9.8~11）；新作物入表必须按公式校验（起步 6 作物表见 mechanics.md §9.3，小萝卜 2 天熟保证首次游玩闭环）；
- 游戏日收入量级：新档前 3 日 **30~80 g/日**（≈12 格耕地）；M1 毕业（Lv5、≈24 格、T2/T3 作物）**200~350 g/日**；
- 消费档位（约 2.5×/档）：工具升级 500g → 2,500g（即时生效）；背包 12→24 格 1,000g；建筑（M3）2,000g → 6,000g → 15,000g，工期 2 游戏日；
- 农场等级 1~10 累计 XP 阈值：**100 / 380 / 770 / 1,300 / 2,150 / 3,300 / 4,800 / 6,900 / 10,000 / 15,000**；收获 XP ≈ `⌊16 × ln(0.018 × sellPrice + 1)⌋`；
- 浇水规则：每日一浇、不浇停滞不死、雨天豁免；M1 不做体力、不做品质（品质 M3 引入：银 1.25×、金 1.5×）。

### 4.5 操作方式（键鼠为主）
- 目标平台：**桌面浏览器**（唯一）；移动：WASD / 方向键（四向）；
- 交互：**E 键作用于面朝格**，或鼠标左键点击目标格——两者等价、全程可混用；
- 工具/物品栏：数字键 **1~9** + 滚轮切换；**Esc** 打开菜单（即暂停）；日结算屏任意键/点击关闭；
- 手柄与触屏不在 M1~M5 范围。

### 4.6 美术与音频素材（选定）
- **主方案：Kenney CC0**——Roguelike/RPG pack（16×16，1700+ tiles）+ Tiny Town（16×16）做地形/建筑/UI；音效 Kenney Audio（UI Audio / Impact Sounds）；BGM 用 FreePD CC0 GitHub 镜像；
- 理由：唯一可整体 commit 进公开仓库的体系（CC0：可商用、可再分发、零署名义务），安装零摩擦，符合「开源产品标准」与「本地与隐私优先」；
- Sprout Lands / Sunnyside / Cozy Farm 仅作风格参考（许可禁再分发，**不得入仓库**）；**绝不使用星露谷本体或提取素材（红线）**；
- `packages/game/assets/manifest.json` 逐文件记录作者/来源/许可/redistributable；未来升级观感（LPC 32×32 整体切换，或取得 Sunnyside 书面分发许可）均属修宪事项。

## 5. 里程碑内容边界（M1~M5）

| 里程碑 | 包含（In） | 明确不做（Out） |
|---|---|---|
| **M1 农场核心循环** | 1 张农场地图（≤64×64）；四向移动；锄地→播种→浇水→收割→出售完整闭环；6 种作物（T1~T3）；雨天豁免；金币 + 农场 Lv1~5；工具 3 档；日循环 + 日结算屏；IndexedDB 存档 + JSON 导入导出 | 体力、品质、动物、建筑、NPC、季节轮替、钓鱼/挖矿/战斗、HUD |
| **M2 会话 HUD** | daemon 雏形（127.0.0.1:43110）；hooks 幂等安装器；三源仲裁状态机（hooks > transcript > ps）；WS 推送；左上角面板（working/blocked/done/idle/unknown 五态显示）；开工第一件事：hook 事件录制器 | 任何干预动作；HUD 与游戏经济的任何绑定；公网/多机 |
| **M3 建造与升级** | 建筑 3 种（鸡舍 2,000g / 加工棚 6,000g / 温室 15,000g）；农场 Lv6~10；Lv5 职业二选一（园艺师 +10% 售价 vs 工匠）；品质系统；洒水器（Lv6 配方）；背包扩容；收集图鉴 | AI 关卡；动物玩法深化（仅鸡舍基础产出） |
| **M4 AI 关卡** | headless `claude -p` 生成管线（≤$0.20/次）；NPC 1~3 名；decision / reflection 两类任务；同屏未处理任务 ≤1、全局生成 ≤1 个 / 15 真实分钟；思考笔记落盘 `~/.codestead/notes/`；游戏内奖励（金币/XP/道具）；总开关、默认克制、可整体关闭 | 任务强制弹窗/打断；笔记自动回填会话（仅留接口）；任何形式的上传 |
| **M5 开源发布** | `npx codestead` 单进程（daemon 同端口托管游戏产物）；中英 README 与安装文档；CI（ubuntu+macos × Node 22/24）；Playwright 冒烟；CREDITS 与许可审计；Windows 支持补位 | 云服务、账号体系、遥测（永久 Out） |

## 6. 统一术语表（中英对照）

| 中文 | English | 约定 |
|---|---|---|
| 会话 | session | 一个运行中的 Claude Code 实例 |
| 四态 | working / blocked / done / idle | blocked=等待用户输入；done=完成未查看；另有展示用 unknown（未装 hooks） |
| 关卡 / 任务 | quest | AI 即时生成的 NPC 任务/对话，kind ∈ {decision, reflection} |
| 会话状态面板 | session HUD | 左上角常驻，只展示不干预 |
| 守护进程 | daemon | 本机 Node 进程：会话监控 + WS 推送 + 关卡生成 |
| 模拟层 | sim layer | `game/src/sim/`，纯 TS、零 Phaser 依赖、确定性 tick |
| 游戏日 | game day | 6:00–22:00，3 真实分钟 |
| 日结算屏 | day summary | 22:00 自动触发的放下点 |
| 明日之诺 | tomorrow's promise | 每次游玩结束时至少有一件「明天会发生的事」 |
| 放下点 | put-down point | 设计出来的自然离开时机（日结算屏为主） |
| 作物档位 | crop tier (T1~T3) | 按 §4.4 公式定价 |
| 再生作物 | regrowing crop | 首熟后按 regrowDays 重复收获 |
| 耕地 | tilled tile | 锄过、可播种的格子 |
| 农场等级 | farm level | 1~10，XP 阈值见 §4.4 |
| 职业 | profession | Lv5 二选一，不可兼得 |
| 收集图鉴 | collection log | 每种作物首次售出点亮 |
| 思考笔记 | thinking note | 玩家对 quest 的作答，仅落盘本机 |
| 零焦虑承诺 | zero-anxiety promise | 现实时间不驱动任何游戏状态 |

## 7. 验收红线（每个里程碑出厂前自检）

1. 新档前 10 分钟内完成一次完整「种→收→卖」并升到农场 Lv2；
2. 任意 2 分钟游玩：至少浇完水 + 看到一个进度条前进；
3. 任意离开：不存在任何会随真实时间恶化的状态；
4. HUD 与 quest 全程未产生一次强制打断；
5. 仓库内全部资产可在 manifest.json 中追溯到 CC0 / 可再分发许可。
