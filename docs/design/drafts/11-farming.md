## 0. 三支柱自检（宪法 §3）

| 支柱 | 本稿如何满足 |
|---|---|
| 游戏自身成立 | 浇水/收获/播种全部产生即时 XP 与金币推进；任意 2 分钟游玩至少「浇完水 + XP 条前进」；日结算屏由耕作数据喂出「明日之诺」（明天什么成熟、哪茬可再收） |
| 零焦虑承诺 | 不浇水只停滞不死；成熟作物无限期留在地里不腐烂；枯萎**只**由游戏内换季事件触发（M1 永久春季，不可达）；模拟层禁用一切真实时钟 |
| 感知不干预 | 耕作系统与会话 HUD / quest 零耦合；日结算屏的会话状态展示由 HUD 子系统注入，本系统只提供版面槽位 |

## 1. 地块模型与状态机

### 1.1 数据模型

每个可耕格（map 图层属性 `farmable=true`）由三个正交维度组成，避免状态组合爆炸：

```ts
// packages/game/src/sim/farm/types.ts —— 纯 TS，零 Phaser 依赖（tech-stack §1 sim 层纪律）
export type GroundKind = 'grass' | 'tilled';

export interface CropState {
  cropId: CropId;
  stage: number;        // 0..stageDays.length-1；成熟后无意义
  daysInStage: number;  // 当前阶段已生长天数
  mature: boolean;
  regrowLeft?: number;  // 仅再生作物收获后存在；0 = 下个结算变回 mature
  withered: boolean;    // M3 换季事件置位；M1 不可达
}

export interface TileState {
  ground: GroundKind;
  wet: boolean;             // 仅 ground === 'tilled' 时有意义
  wateredXpToday: boolean;  // 当日该格是否已发放浇水 XP（防重复）
  everTilled: boolean;      // 开垦 XP 只发一次
  crop?: CropState;
}
```

存档采用稀疏表：仅序列化非默认格（`ground !== 'grass' || crop` 存在），schema 定义在 `packages/shared/src/save.ts`（zod，含版本迁移）。

### 1.2 状态机总图

```
              锄头(E/点击)
   [草地] ───────────────► [耕地·干]
                             │   ▲
                       水壶  │   │ 日结算回干（明日非雨）
                             ▼   │
                           [耕地·湿]
                             │
        播种（干/湿均可，消耗种子×1）
                             ▼
            ┌────────[生长 stage 0..N-1]────────┐
            │ 每个「已浇水的夜晚」推进 1 天        │
            │ 当日未浇水：停滞（不死、无惩罚）      │
            └───────────────┬───────────────────┘
                            ▼ stageDays 走完
                       [成熟 mature] ◄────────────────┐
                        │        │                    │
              E/点击收获 │        │ 换季且新季∉seasons │ regrowLeft 减到 0
                        │        │ （M3 事件）        │ （每个浇水日 −1）
        ┌───────────────┘        ▼                    │
        │ 单次收获作物        [枯萎 withered]    [再生中 regrowing]
        │ → 作物移除，          │                     ▲
        │   回到[耕地·干/湿]    │ 镰刀/锄头清除        │
        │ 再生作物              ▼                     │
        └──────────────► [耕地·干]      收获(再生作物)─┘
```

### 1.3 转移表（条件 × 工具 × 产出）

| # | 从 | 到 | 触发 | 条件 | 工具/消耗 | 产出 |
|---|---|---|---|---|---|---|
| T1 | 草地 | 耕地·干 | E / 点击 | `farmable=true`、无障碍物、无作物 | 锄头 | 该格首次开垦 +1 XP |
| T2 | 耕地·干 | 耕地·湿 | E / 点击 | — | 水壶（M1 无水量限制） | 格上有生长中作物时 +1 XP/格·日 |
| T3 | 耕地（干/湿） | 已播种 stage 0 | E / 点击 | 无作物；作物 `seasons` 含当前季；`farmLevel ≥ unlockLevel`；背包有该种子 | 种子 ×1 | +1 XP；播种时若格已湿，当日即计入「已浇水」 |
| T4 | 生长 stage i | stage i+1 或 成熟 | 22:00 日结算 | 当日该格 `wet === true`（手动浇 / 雨天 / M3 洒水器） | — | — |
| T5 | 生长 / 再生中 | 原状态（停滞） | 22:00 日结算 | 当日未浇水 | — | 无任何惩罚 |
| T6 | 成熟（单次） | 作物移除，耕地保持 | E / 点击 | 背包有空位或同类未满堆叠 | 任意（收获优先级最高，见 §3.4） | 产物 ×1 + 作物 XP |
| T7 | 成熟（再生） | 再生中（`regrowLeft = regrowDays`） | E / 点击 | 同 T6 | 任意 | 产物 ×1 + 作物 XP（每茬） |
| T8 | 再生中 | 成熟 | 22:00 日结算 | 每个浇水日 `regrowLeft − 1`，减到 0 后下次结算转成熟 | — | — |
| T9 | 生长 / 再生中 | 作物移除 | E / 点击 | — | 镰刀（铲除，不退种子） | — |
| T10 | 生长 / 成熟 / 再生中 | 枯萎 | 换季瞬间（M3 游戏内事件） | 新季节 ∉ `seasons` | — | 前一天日结算屏温和预告（见 §9 边界 11） |
| T11 | 枯萎 | 耕地·干 | E / 点击 | — | 镰刀或锄头 | — |
| T12 | 耕地·湿 | 耕地·干 | 22:00 日结算 | 明日非雨 | — | — |
| T13 | 全部耕地·干 | 耕地·湿 | 雨天 6:00 自动 | 当日天气 = 雨 | — | 「假日惊喜」，免浇水 |

**刻意不做的转移**（M1 决策，防误操作与零焦虑）：

- 空耕地**不会**自动退化回草地（无「时间惩罚」、不打扰离开很久的玩家）；M3 再评估「铲子」工具用于美化重整；
- 成熟单次作物**永不**过期/腐烂，可无限期留在地里等收；
- 锄头作用于已耕地或有作物的格 = no-op（保护作物，不会误铲）。

### 1.4 日结算处理顺序（确定性，sim 层可单测）

```
22:00 自动入睡 →
  1. 对每个有生长中/再生中作物的格：若 wet === true → 推进（T4/T8）
  2. 统计今日数据 → 喂给日结算屏（收成、收入、明日之诺）
  3. 所有耕地 wet = false、wateredXpToday = false（T12）
  4. 读取明日天气 weather[day+1]（时间子系统按存档种子预生成，预报 100% 准确）
  5. 若明日 = 雨：6:00 时所有耕地 wet = true（T13）
  6. 存档落盘（IndexedDB，宪法 §4.3）
→ 显示日结算屏（停留不限时）→ 任意键 → 次日 6:00
```

## 2. 浇水与天气

| 规则 | 规格 | 依据 |
|---|---|---|
| 频率 | 每株生长中/再生中作物每游戏日浇一次 | mechanics §2 |
| 漏浇后果 | **当日不生长（停滞），绝不枯死、绝无惩罚**；漏 N 天 = 成熟推迟 N 天 | 宪法 §4.4「不浇停滞不死」 |
| 成熟单次作物 | 不需要浇水，可留在地里等收 | mechanics §2 |
| 成熟再生作物 | 收获后的「再生中」阶段需每日浇水推进 `regrowLeft` | mechanics §2 |
| 雨天 | 当日 6:00 全部耕地自动湿润，免浇水（雨天手动浇水 = no-op，无 XP） | 宪法 §4.4「雨天豁免」 |
| 雨概率（接口假设，归时间子系统） | 春 25% / 夏 10% / 秋 20%，按存档种子预生成整季序列；**新档第 1~2 天强制晴**（保证浇水教学） | 本稿提案 |
| 预报 | 明日天气在日结算屏 100% 准确展示（「明日之诺」的一部分） | 宪法 §4.3 日结算屏 |
| 水壶水量 | **M1 无限水，无补水动作**（砍掉一套无聊跑腿；M3 若加体力/水量再评估） | 本稿决策 |

## 3. 工具

### 3.1 清单与升级（宪法 §4.4：500g → 2,500g，即时生效）

| 工具 | 用途 | 木（初始） | 铜 500g | 金 2,500g |
|---|---|---|---|---|
| 锄头 hoe | 草地 → 耕地；清枯萎 | 1 格 | 直线 3 格 | 3×3 |
| 水壶 watering_can | 湿润耕地 | 1 格 | 直线 3 格 | 3×3 |
| 镰刀 sickle | 铲除生长中作物 / 清枯萎 / 收获 | 1 格（M1 不升级） | — | — |

- 升级在集市板（§7.4）购买，**即时生效**（无 SDV 式「送修 2 天」，宪法已裁决）；锄头与水壶**分别**购买升级；
- 「直线 3 格」= 面朝格 + 面朝方向再延伸 2 格；「3×3」= 以面朝格为中心的 9 格。

### 3.2 范围工具的触发方式（防误操作）

- **轻按 E / 单击**：永远只作用 1 格（面朝格 / 点击格）；
- **长按 E / 按住左键 ≥ 0.4s**：显示范围高亮预览，松开时对范围内**所有合法目标格**生效（非法格自动跳过，XP 按实际生效格数计）；
- 设计理由：金锄头 3×3 若默认全开，会把不想开垦的格子误锄开（M1 没有「取消开垦」），轻按/长按双模式兜住误操作。

### 3.3 交互与目标选择（宪法 §4.5）

- E 键作用于**面朝格**（四向移动的朝向相邻格）；
- 鼠标左键点击：目标格必须在玩家周身 3×3 范围内（含脚下格），范围外点击 = no-op + 光标灰显；不做寻路走过去（M1）；
- 面朝格/点击格永远渲染高亮框：绿色 = 当前选中物品可作用，灰色 = 不可作用；
- 数字键 1~9 + 滚轮切换物品栏；Esc 菜单即暂停（时间即停）。

### 3.4 同格动作优先级（按当前选中物品裁决）

| 选中物品 | 面朝格状态 → 动作 |
|---|---|
| （任意） | **成熟作物 → 收获**（最高优先级，任何工具在手都先收获，防误铲） |
| 锄头 | 草地 → 开垦；枯萎 → 清除；其余 → no-op |
| 水壶 | 耕地·干（含有作物）→ 浇水；其余 → no-op |
| 种子 | 无作物的耕地 → 播种；其余 → no-op |
| 镰刀 | 生长中/再生中作物 → 铲除（不退种子）；枯萎 → 清除 |
| 农产品/其他 | no-op |

## 4. 作物数值表（15 种）

### 4.1 定价口径（宪法 §4.4 公式的执行细则）

- **单次收获作物（严格受档位带约束）**：`goldPerDay = (sellPrice − seedPrice) / growthDays`，必须落在档位带内：**T1 = 4.0（±0.1）｜T2 = 6.5~7.5｜T3 = 9.8~11**；
- **再生作物（headline = 满季口径）**：第 1 天播种、每日浇水、收满整季（28 天）：`goldPerDay = (茬数 × sellPrice − seedPrice) / 27`。再生作物的设计意图是「奖励长期承诺」（mechanics §3.2 法则 3）：满季口径**允许超出名义档位带**，上限为 **T2 再生 ≤ 9.0、T3 再生 ≤ 12.0**；晚种则收益骤降（见表内区间列），整体期望仍贴近名义档位；
- 产量：M1 全部 `expectedYield = 1`，无随机多收（确定性 sim 优先；随机产量与品质 M3 一并评估）；
- XP：新作物按宪法公式 `⌊16 × ln(0.018 × sellPrice + 1)⌋` 严格计算；**M1 六作物的 XP 沿用 mechanics §9.3 原值**（宪法明文指认该表），与公式偏差 ≤ +5，已逐一标注。

- **小萝卜（2 天熟）**是教学作物：保证新档前 10 分钟完成完整「种→收→卖」闭环（红线 1，推演见 §6）；
- 每季都有一个 Lv1 可用的 T1 作物（小萝卜/辣椒/小麦），换季永远不会「无种可种」；
- 再生茬数推导（第 1 天播种）：豆藤首熟第 9 天、之后每 2 天一茬 → 9,11,…,27 共 10 茬；浆果 9,12,…,27 共 7 茬；番茄 8,11,…,26 共 7 茬；茄子 7,10,…,28 共 8 茬；蔓越莓 8,10,…,28 共 11 茬；
- M1 永久春季（换季 M3 启用）：豆藤/浆果事实上「无限再生」，作为 Lv3/Lv5 的解锁奖励是刻意的——其满季口径（8.9 / 11.9）仍被卷心菜（9.8 单次、可随时改种）和后续内容压制。

### 4.3 数据结构（`packages/game/src/sim/crops/cropDefs.ts`）

```ts
export type Season = 'spring' | 'summer' | 'fall';

export interface CropDef {
  id: CropId;
  nameKey: string;          // i18n key, e.g. "crop.radish_quick"
  tier: 1 | 2 | 3;
  seasons: Season[];        // sunflower: ['summer','fall']
  seedPrice: number;
  sellPrice: number;        // 每茬/每个；M3 品质乘数由经济子系统在售出时套用
  growthDays: number;       // === stageDays 之和（cropDefs 单测断言）
  stageDays: number[];      // 每生长阶段天数
  regrowDays?: number;      // 再生作物收获后到下次成熟的浇水日数
  xpHarvest: number;        // 每茬收获 XP
  unlockLevel: number;      // 农场等级解锁门槛
  spriteRow: number;        // 作物图集行号（见 §5）
}
```

校验单测（vitest，sim 层）：每个 `CropDef` 自动断言 ① `sum(stageDays) === growthDays`；② 单次作物 goldPerDay 落在档位带；③ 再生作物满季口径 ≤ 档位上限（T2≤9.0 / T3≤12.0）；④ M3 作物 `xpHarvest === floor(16*ln(0.018*sellPrice+1))`。

## 5. 作物视觉阶段（美术规格）

- 视觉帧数 = `stageDays.length + 1`（最后一帧 = 成熟）；**第 0 帧统一使用共享的「种子土丘」贴图**（全作物共用 1 帧，省美术）；
- 再生作物额外 +1 帧「已收割再生中」（成熟体去掉果实，如无豆荚的豆藤）；
- 枯萎使用**全作物共享**的 1 帧通用枯草贴图（M3 才可达，M1 不出资产）；
- 帧规格：16×16，作物图集每行一种作物（`spriteRow`），列 = 阶段；阶段切换发生在 6:00（玩家醒来看到变化，强化过夜回报）；
- 成熟帧必须与前一帧有显著轮廓差异（可收割性一眼可辨），并叠加 1px 浅色描边高亮（渲染层处理，非美术帧）；
- 资产来源：Kenney CC0 体系（assets.md 主方案）；Kenney 现成作物帧不足时按其调色板自绘补帧，同样 CC0 入 `manifest.json`。

## 6. XP 来源与开局红线推演

### 6.1 耕作 XP 来源（本稿新增设计）

| 动作 | XP | 防刷约束 |
|---|---:|---|
| 开垦（锄草地） | +1 | 每格**终身一次**（`everTilled` 标记） |
| 播种 | +1 | 消耗种子，金币天然约束 |
| 手动浇水 | +1/格·日 | 仅对**有生长中/再生中作物**的格；每格每日一次（`wateredXpToday`）；雨天/已湿格 no-op 无 XP；M3 洒水器**不**产 XP（自动化以 XP 换便利） |
| 收获 | +`xpHarvest`/茬 | — |

浇水 XP 的设计意图：让「每日仪式」本身可见地推进 XP 条，保证红线 2（任意 2 分钟游玩 ≥ 一个进度条前进）在不跨夜的短会话里也成立。

### 6.2 等级阈值的执行口径（需修宪确认，见开放问题 Q1）

- 新档 = **农场 Lv1、0 XP**；宪法累计阈值表依序映射为**升到下一级**所需：Lv2=100、Lv3=380、Lv4=770、Lv5=1,300、Lv6=2,150、Lv7=3,300、Lv8=4,800、Lv9=6,900、Lv10=10,000；表尾 15,000 保留为「完美农场」成就线（M5 评估）；
- 若按字面「Lv1 需 100 XP（即新档 Lv0）」执行，红线 1「前 10 分钟升到 Lv2」需要 380 XP，推演证明不可达——故采用本口径并提请修宪明确。

### 6.3 红线 1 推演（新档前 10 分钟 ≈ 3.3 游戏日）

前提：初始 100g（宪法）、初始工具木锄/木壶/镰刀、背包 12 格、第 1~2 天强制晴。

| 游戏日 | 玩家动作 | 金币 | 累计 XP |
|---|---|---:|---:|
| 第 1 天 | 开垦 12 格（+12）；集市板买 10 包小萝卜种（−100g）；播种 10（+10）；浇水 10（+10） | 0 | 32 |
| 第 2 天 | 浇水 10（+10）；（剩余时间探索地图） | 0 | 42 |
| 第 3 天 | 6:00 小萝卜 ×10 成熟；收获（+5×10=50）→ **92**；集市板卖 10×18g（+180g）；买 12 包种（−120g）；播种 12（+12）→ **104 ≥ 100，升 Lv2** ✅；浇水 12（+12） | 60 | 116 |

- 完整「种→收→卖」于第 3 天闭环（≤9 分钟）✅；农场 Lv2 于第 3 天达成 ✅；
- 前 3 日收入 180g / 3 天 = **60g/日**，落在宪法「新档前 3 日 30~80g/日」区间 ✅；
- Lv2 解锁土豆（T2），立即给出下一个「明日之诺」。

### 6.4 M1 毕业校验（宪法 §4.4）

Lv5、24 格、T2/T3 混种：24 格卷心菜 ≈ 24×9.8 ≈ **235g/日**；24 格浆果（满茬期）≈ 24×11.9 ≈ **286g/日**——均落在「200~350g/日」区间 ✅。日 XP 吞吐 ≈ 浇水 24 + 收获摊薄 ≈ 75~85 XP/日，Lv2→Lv5（1,300 XP）约 15~18 游戏日 ≈ 45~55 真实分钟，符合 M1 体量。

## 9. 边界情况清单

1. **背包满时收获**：动作阻止、作物保持成熟（永不腐烂）、提示「背包已满」——绝不掉落销毁产物；
2. **浇已湿的格 / 雨天浇水**：no-op，不发 XP（`wateredXpToday` + wet 双重防刷）；
3. **范围工具部分越界/非法**：只作用合法子集，XP 按实际生效格计；全部非法 = no-op；
4. **22:00 强制入睡打断操作**：长按蓄力中的范围预览直接取消、不生效；无任何惩罚（宪法：无晕倒）；
5. **镰刀铲除生长中作物**：立即生效、不退种子；因需要主动切换到镰刀（§3.4 优先级保护成熟作物），不再加确认弹窗；
6. **空耕地永不退化**；锄头对耕地 no-op（无「取消开垦」，M3 再议铲子）；
7. **再生作物在「再生中」漏浇**：`regrowLeft` 停滞不减，永不死亡；
8. **播种当日即浇过的湿格**：当日计为已浇水（T3 备注），新手少踩一脚；
9. **同格重复播种 / 对有作物格播种**：阻止；
10. **金币不足/等级不足**：集市板条目置灰可见，标注差额；
11. **M3 换季枯萎**：仅发生在换季瞬间（游戏内事件，永不由真实时间触发）；换季前一天的日结算屏温和预告「X 株作物将随换季枯萎」，给玩家一次抢收机会；M1 永久春季，该路径不可达但 sim 层带测试实现；
12. **满级（Lv10）后**：XP 继续累计（为 15,000 完美线保留），不再升级；
13. **存档/读档**：`TileState` 全量往返（含 wet、`daysInStage`、`regrowLeft`），读档后当日浇水状态不丢；
14. **跨日界游玩**（2 分钟会话未跨 22:00）：浇水 XP 保证仍有进度条前进（红线 2）；
15. **向日葵双季（夏/秋）**：夏季种的向日葵跨入秋季不枯萎（`seasons` 含秋）——`withered` 判定用集合包含而非相等。

## 10. 验收标准（sim 层 vitest「快进 N 天」测试，全部可代码化）

1. **红线 1 脚本回放**：按 §6.3 的动作序列驱动 sim，断言第 3 游戏日结束时 `gold ≥ 60 && xp ≥ 100 && level ≥ 2`，且已发生 ≥ 1 次完整「种→收→卖」；
2. **档位带校验**：cropDefs 单测——单次作物 goldPerDay ∈ 档位带；再生作物满季口径 ≤ T2 9.0 / T3 12.0；`sum(stageDays) === growthDays`；M3 作物 XP === 公式值；
3. **停滞不死**：种植后快进 30 天不浇水——作物 stage 永远为 0、永不枯死；之后连续浇 `growthDays` 天必然成熟；
4. **雨天豁免**：雨日不浇水，作物照常推进；雨日手动浇水产生 0 XP；
5. **确定性**：同一存档种子 + 同一动作日志，快进 28 天后 farm 状态哈希逐字节一致（连跑 100 次）；
6. **零真实时间**：`sim/**` 内 `Date.now / setTimeout / performance.now` 出现次数 = 0（ESLint `no-restricted-globals` + grep CI 检查）；
7. **背包满阻止收获**：填满 12 格异类物品后收获 → 作物仍 mature、无物品丢失；
8. **工具范围**：铜壶单次动作湿润恰好 ≤ 3 格直线、金壶 ≤ 9 格；非法格跳过且 XP 计数 = 实际生效格数；
9. **再生节奏**：第 1 天种豆藤、每日浇水，断言收获日恰为 9,11,13,…,27（共 10 茬）；浆果 9,12,…,27（7 茬）；
10. **M1 毕业模拟**：贪心 bot（始终种当前等级最优作物、24 格）快进至 Lv5，断言毕业期日收入 ∈ [200, 350]g；
11. **换季枯萎（M3 预埋）**：春→夏切换时春季独占作物 `withered === true`、向日葵跨夏/秋不枯；枯萎仅由 `onSeasonChange` 触发；
12. **防刷**：同格重复开垦/重复浇水/雨天浇水 → XP 增量为 0。
