## 0. 与宪法三支柱逐条对照（自检）

| 支柱 | 本系统如何满足 |
|---|---|
| 游戏自身成立 | 建造是「明日之诺」最强发生器：付费下单 → 工地 2 游戏日 → 竣工剪彩。每个图纸都解锁真实玩法（产蛋/加工/全季种植/收纳），不是数值皮肤；装饰物提供零目的的自我表达（候车室式冥想）。 |
| 零焦虑承诺 | 工期只按**游戏内日**推进，关页即停、零损失；搬迁永久免费；装饰与小设施拆除 100% 返还；工地随时取消全额退款；建造期间农舍照常可用（不学 SDV 锁门）；无任何「建错了被惩罚」的状态。 |
| 感知不干预 | 建造系统与会话 HUD / quest 零耦合；竣工只在日结算屏与场景内安静呈现，永不弹窗打断。农舍升级 II 的「书房」仅为 M4 思考笔记提供**被动陈列**接口，不主动提醒。 |

里程碑归属：整个系统在 **M3** 实装（宪法 §5）；M1/M2 不含任何建造代码。宪法 M3 In 列表写明「建筑 3 种」——本稿口径：**「建筑（building）」专指鸡舍/加工棚/温室三种大型建筑**，农舍升级链、小型设施（station）与装饰物（decoration）不计入「建筑」数目，但同属本建造系统。该口径是否成立见 §13 待决问题 Q1。

## 1. 系统概览

建造系统让玩家把金币与材料转化为**空间上的永久进度**。三类可放置物（placeable）：

| 类别 | 英文 | 特征 | 例 |
|---|---|---|---|
| 大型建筑 | `building` | 占多格、有工期（2 游戏日）、有室内场景、限 1 座（除非另注） | 鸡舍、加工棚、温室 |
| 小型设施 | `station` | 占 1~2 格、即时建成、有功能交互 | 仓库箱、烘干架、（洒水器*） |
| 装饰物 | `decoration` | 占 1~4 格、即时建成、纯观感/轻交互 | 栅栏、石径、花坛、长椅、灯柱 |

\* 洒水器（Lv6 配方）的**功能数值属 farming 子系统**，但其放置/拆除/搬迁复用本系统的 placeable 管线（见 §12 接口 I-2）。

农舍（farmhouse）是地图固有建筑，不可建造、不可搬迁、不可拆除，但有**升级链**（升级 I / 升级 II），走与大型建筑相同的「下单 → 工期 → 竣工」流程。

### 1.1 材料系统（最小集：两种）

| 材料 id | 名称 | 获取途径 1（采集） | 获取途径 2（商店兜底） | 备注 |
|---|---|---|---|---|
| `wood` | 木材 | 斧清理农场上的树/树桩，5 木/棵 | 5g/个 | 占背包 1 格，单格堆叠 99 |
| `stone` | 石料 | 镐清理石块，3 石/块 | 5g/个 | 同上 |

- 农场地图初始资源量假设：约 40 个树点（≈200 木）+ 30 个石点（≈90 石）；每游戏日清晨在地图边缘带再生 2~4 个资源点（≈10 木 + 6 石/日）。再生按**游戏日**计，不玩不刷（零焦虑）。
- 采集工具（斧/镐）与资源点刷新规则的最终数值归 farming/map 子系统（接口 I-1、I-6）；商店兜底价 5g 归 economy 子系统确认（接口 I-3）。本稿所有材料成本按「1 木/石 ≈ 5g 等价金」做经济校验。
- 不引入第三种材料（黏土/矿锭等）：两种已足够制造「金币 + 体力外的第二种稀缺」，再多是无谓复杂度。

## 2. 设施清单与数值总表（12 种）

> 经济校验基准（宪法 §4.4）：M1 毕业（Lv5）日收入 200~350g；建筑档位 2,000 → 6,000 → 15,000（约 2.5×/档）；工期一律 2 游戏日。材料折金按 5g/个 计入「等效总价」。

### 2.1 大型建筑（building，工期 2 游戏日，各限 1 座）

| id | 名称 | 占格 (宽×高) | 金币 | 材料 | 等效总价 | 解锁 | 功能 |
|---|---|---|---:|---|---:|---|---|
| `coop` | 鸡舍 | 4×3（门在底边中央） | 2,000g | 木×150 | 2,750g | 农场 Lv5 | 容纳 4 只鸡，附赠 2 只；每鸡每游戏日清晨产 1 枚蛋（蛋售价 40g）；加购鸡 400g/只（商店）；**免喂养、动物不会死**（宪法 M3「动物玩法深化 Out」+ 零焦虑） |
| `workshop` | 加工棚 | 5×3（门在底边中央） | 6,000g | 木×200 石×100 | 7,500g | 农场 Lv7 | 室内 6 个加工槽。配方：果酱（任意作物 ×1 → 售价 = 2×作物售价 + 25，工时 2 游戏日）；蛋黄酱（蛋 ×1 → 95g，工时 1 游戏日）。工匠职业加成作用于加工品售价（系数见 Q3） |
| `greenhouse` | 温室 | 6×5（门在底边中央） | 15,000g | 木×300 石×300 | 18,000g | 农场 Lv9 | 室内 24 格固定耕地，无视季节（换季规则启用后即全季可种）；室内无雨天豁免，需浇水或洒水器；M3 毕业级金币汇 |

回本估算（验证定价合理）：

- 鸡舍：2 鸡 80g/日，满 4 鸡 160g/日 → 等效 2,750g 约 17~34 游戏日回本（叠加蛋黄酱后约 12 日），Lv5 后的中线投资，合理；
- 加工棚：6 槽全开果酱（以卷心菜 178g 计：+203g/2 日/槽 ≈ 101g/槽日）→ 满载 +609g/日，等效 7,500g 约 12 游戏日回本，但受作物供给制约，实际 20 日级，合理；
- 温室：24 格 × T3 ≈ 10g/格日 = +240g/日 → 约 75 游戏日（≈3.7 真实小时游玩）回本，定位为长线毕业目标 + 换季保险，合理。

## 3. 建造模式交互流程

### 3.1 入口与按键

- **B 键**直接打开建造目录（新增按键，宪法 §4.5 未列，见 Q5）；等价入口：**Esc 菜单 → 「建造」页签**（保证不学新键也能用）；
- 全程键鼠等价（宪法 §4.5）：鼠标移动 = 预览跟格，左键 = 确认，右键 = 取消；方向键/WASD = 逐格移动预览光标，E = 确认，Esc = 取消/返回。

### 3.2 状态机

```
                 B / Esc菜单→建造
   ┌────────┐ ───────────────────► ┌──────────────┐
   │ CLOSED │                      │   CATALOG    │  目录页（=菜单，sim tick 全停）
   └────────┘ ◄─────────────────── │ 图纸|搬迁|拆除 │
        ▲          Esc / 关闭       └──────┬───────┘
        │                                  │ 选中可建图纸 / 选中搬迁目标 / 进入拆除
        │                                  ▼
        │                          ┌──────────────┐
        │     Esc / 右键取消        │   PLACING    │  放置模式（在世界中，时间恢复流动）
        ├───────────────────────── │  (ghost 预览) │ ◄── MOVING 与 PLACING 共用该态
        │                          └──────┬───────┘     （MOVING 多带 sourceInstanceId）
        │                                 │ E / 左键 且 校验通过
        │                                 ▼
        │                          ┌──────────────┐
        │      建筑类弹确认对话框    │   CONFIRM    │  仅 building/房屋升级（=对话，tick 停）
        │  ◄──「取消」──            │ （金额+工期） │  station/decoration 跳过此态直接成交
        │                          └──────┬───────┘
        │                                 │ 确认
        │                                 ▼
        │                          ┌──────────────┐
        └───────────────────────── │  COMMITTED   │  扣费扣料 → 生成工地/实体
          station/decoration:       └──────────────┘
          停留在 PLACING 连续放置          │ building: 工地 underConstruction(daysLeft=2)
          （栅栏/石径刷格体验）            │ 过夜 daysLeft-1 → 0 时竣工（清晨 6:00 换贴图）
```

状态迁移表（实现 `game/src/sim/build/buildMode.ts`，纯函数 reducer，与渲染解耦）：

| 当前态 | 事件 | 守卫条件 | 次态 | 副作用 |
|---|---|---|---|---|
| CLOSED | OPEN_CATALOG | — | CATALOG | 暂停 sim tick |
| CATALOG | SELECT_BLUEPRINT(id) | 已解锁 且 余额/材料足 且 未达上限 | PLACING | 恢复 tick；ghost 置于玩家面朝格 |
| CATALOG | SELECT_MOVE(instanceId) | def.movable | PLACING(moving) | ghost = 该实体；原占格临时豁免碰撞 |
| CATALOG | SELECT_DEMOLISH | — | CATALOG(demolish 子页) | 光标变拆除模式 |
| PLACING | CURSOR_MOVE(x,y) | — | PLACING | 重算 footprint 校验，刷新红绿着色 |
| PLACING | CONFIRM | canPlace() == true | building/升级 → CONFIRM；其余 → COMMITTED | — |
| PLACING | CONFIRM | canPlace() == false | PLACING | 抖动 + 提示音（无惩罚） |
| PLACING | CANCEL | — | CATALOG | moving 时实体原位还原 |
| PLACING | DAY_END(22:00) | — | CLOSED | 强制退出（未确认=未扣费），进日结算屏 |
| CONFIRM | ACCEPT | 余额复验通过 | COMMITTED | 扣金币+材料；spawn 工地 |
| CONFIRM | CANCEL | — | PLACING | — |
| COMMITTED | （station/decoration） | — | PLACING | 余额仍足→连续放置；不足→CATALOG |
| COMMITTED | （building/升级） | — | CLOSED | 相机短暂聚焦工地 1 秒（非强制锁定） |

时间规则（宪法 §4.3）：CATALOG 与 CONFIRM 属菜单/对话 → tick 全停；PLACING 在世界中 → 时间正常流动（与挥锄头同级的世界内操作）。

### 3.3 放置预览与校验着色

- ghost 以 50% 透明度渲染目标贴图 + 脚印格着色：**绿色半透明 = 可放，红色 = 冲突格**（逐格标红，玩家能看出是哪一格挡住了）；
- 门格以 `D` 标识并要求门前 1 格可通行；
- 校验逐帧只在光标格变化时重算（64×64 地图全量 footprint 检查 < 30 格，无性能问题）。

### 3.4 工期与竣工

- `building` 与房屋升级：确认后原地生成**工地实体**（脚手架贴图），`daysLeft = 2`；每次过夜（22:00 入睡 → 新一天）`daysLeft -= 1`；归零时当日清晨 6:00 换为成品贴图并在场景内冒一个小彩带粒子（无弹窗）；
- 日结算屏逐日出现一行：「鸡舍 还差 1 天完工」→ 次日「鸡舍 竣工！」（接口 I-5），这是建造系统给「明日之诺」的固定供给；
- 工地期间：占格与成品一致、阻挡通行；**随时可取消**（拆除页选中工地 → 全额返还金币与材料，零焦虑）；
- `station` / `decoration`：无工期，确认即建成。

## 4. 放置规则（canPlace 规格）

```
canPlace(def, origin, world) 为 true 当且仅当 footprint 内每格满足：
  1. 在地图边界内；
  2. Tiled 地图该格 buildable == true（由地图 "meta" 对象层属性烘焙；水面、
     河岸、出生点保护区、固定农舍及其门前 2 格均为 false）；
  3. 非耕地/非作物格（farming 查询，接口 I-1）—— 石径同样不可压耕地；
  4. 不与任何已放置 placeable 的 footprint 重叠（搬迁时豁免自身原占格）；
     例外：decoration 可放在石径上（石径属 ground overlay 层，不算占格）；
  5. 玩家与（M3 的）鸡等实体当前不站在 footprint 内；
且对 building 额外要求：
  6. doorOffset 指向的门格之外、正前方 1 格亦满足条款 1~4（保证门可达）。
```

### 4.1 拆除（demolish）

| 类别 | 可拆？ | 返还 | 确认流程 |
|---|---|---|---|
| decoration | 可 | **100% 材料与金币** | 即拆即返，无对话框（摆错零成本，鼓励反复布置） |
| station | 可（仓库箱须为空、烘干架在制品先退回背包） | **100%** | 即拆即返；背包放不下退回物 → 拒绝拆除并提示 |
| building（成品） | 可 | **50% 金币 + 50% 材料**（向下取整） | 二次确认对话框，明示返还额（防误拆重大资产） |
| building（工地） | 可（=取消订单） | **100%** | 单次确认 |
| 农舍及其升级 | 不可拆、不可降级 | — | 目录中不出现拆除项 |

### 4.2 搬迁（move）

- 所有 `movable: true` 的 placeable（三大建筑、仓库箱、烘干架、全部装饰）**永久免费、即时搬迁**；农舍不可搬迁；
- 搬迁保留全部内部状态：仓库箱内容、烘干架在制品与剩余工时、鸡舍内的鸡、温室内的耕地与作物（室内是独立场景，整体随建筑走，作物不受影响）；
- 工地也可搬迁（不重置 daysLeft）。

### 4.3 渲染与层级（实现注记）

- placeable 渲染在对象层，与玩家做 y-sort；石径渲染在 ground overlay 层（作物之下）；

## 7. 数据结构（`packages/game/src/sim/build/`，纯 TS、零 Phaser 依赖）

```ts
// types.ts —— 标识符全英文；定义表静态导出，便于 headless 快进测试
export type PlaceableCategory = 'building' | 'station' | 'decoration';

export interface BlueprintDef {
  id: string;                            // "coop"
  nameKey: string;                       // i18n key
  category: PlaceableCategory;
  size: { w: number; h: number };        // footprint，单位 tile
  doorOffset?: { x: number; y: number }; // building 必填；相对 origin（左上角）
  cost: { gold: number; wood?: number; stone?: number };
  buildDays: number;                     // building/升级=2；station/decoration=0
  unlock: { farmLevel: number; requires?: string[] };  // farmhouse_2 requires ["farmhouse_1"]
  limit?: number;                        // chest/drying_rack=4；building=1；缺省=无限
  movable: boolean;                      // 农舍升级=false（原地变换，无需搬迁）
  demolishable: boolean;                 // 农舍升级=false
  refundRate: number;                    // decoration/station/工地=1.0；building 成品=0.5
  interiorMapId?: string;                // "coop_interior" 等
}

export interface PlacedStructure {
  instanceId: string;                    // nanoid
  defId: string;
  origin: { x: number; y: number };      // 左上角 tile 坐标
  state: 'underConstruction' | 'built';
  daysLeft?: number;                     // underConstruction 时有效
  data?: StructureData;                  // 按类型的运行时数据（zod 校验入档）
}

export type StructureData =
  | { kind: 'chest'; slots: (ItemStack | null)[] }                 // 24 槽
  | { kind: 'dryingRack'; jobs: { cropId: string; daysLeft: number }[] }   // ≤2
  | { kind: 'coop'; hens: number; eggsReady: number }              // hens ≤4
  | { kind: 'workshop'; jobs: { recipeId: string; inputId: string; daysLeft: number }[] }; // ≤6

export interface BuildModeState {       // UI 态，不入存档
  mode: 'closed' | 'catalog' | 'placing' | 'confirm';
  blueprintId?: string;
  movingInstanceId?: string;            // 搬迁时携带
  cursor?: { x: number; y: number };
  valid?: boolean;
}
```

- 存档：`PlacedStructure[]` 加入 save schema（zod，`packages/shared/src/save.ts`），schemaVersion 升一版并附迁移函数（旧档默认 `structures: []`）；`BuildModeState` 是瞬时 UI 态，**不入档**；
- 过夜结算（与作物生长同一处日推进逻辑）：`advanceDay()` 依次处理 工地 daysLeft、烘干架 jobs、加工棚 jobs、鸡舍产蛋——全部纯函数，支持「快进 30 天」headless 测试（tech-stack §1 测试策略）。

## 8. 边界情况清单（实现必须覆盖）

| # | 场景 | 规则 |
|---|---|---|
| E1 | 放置中到达 22:00 自动入睡 | 强制退出 PLACING（未确认=未扣费，零损失），正常进日结算屏 |
| E2 | CONFIRM 对话框打开时 | tick 已停，不存在「确认途中过夜」；ACCEPT 时复验余额（防快进测试中的竞态） |
| E3 | 玩家自己站在 footprint 内 | 该格标红，不可确认（规则 §4 条款 5），不做「自动推开玩家」 |
| E4 | 搬迁后玩家被新位置压住 | 不可能发生（条款 5 含搬迁校验） |
| E5 | 拆仓库箱但箱内有物 | 拒绝并提示「先取空箱子」；不自动倾倒（防丢东西焦虑） |
| E6 | 拆烘干架/加工棚槽位有在制品 | 在制品按原料退回背包（工时作废不补偿，对话框明示）；背包满则拒绝拆除 |
| E7 | 拆鸡舍但有鸡 | 拒绝并提示「先转移/出售鸡」（出售鸡 200g/只，回收半价，归 economy 确认） |
| E8 | 连续放置栅栏时材料耗尽 | 当前格成交后自动退回 CATALOG，黄字提示缺料；无任何惩罚 |
| E9 | 石径与耕地 | 石径不可压耕地；锄头不可锄石径（需先拆径再锄，提示文案引导） |
| E10 | 洒水器/作物在被搬迁建筑门前 | 建筑搬迁只校验新位置；旧位置释放后地面状态不变 |
| E11 | 温室搬迁时室内有作物 | 室内是独立场景，整体随建筑实例走，作物/浇水状态不变 |
| E12 | 存档导入时 footprint 重叠/越界 | zod 结构校验后追加语义校验：非法实体整体收回——金币与材料 100% 折返给玩家并在载入提示，不静默删除 |
| E13 | 上限校验（chest=4 等） | CATALOG 中达上限项显示「已达上限 4/4」，不可进入放置 |
| E14 | 旋转 | v1 不支持旋转（16×16 素材无旋转帧）；栅栏方向由 auto-tile 自动处理 |
| E15 | 雨天/夜间建造 | 无任何限制（不引入「天气阻工」这类隐性时间惩罚） |
| E16 | B 键在对话/结算屏中按下 | 忽略（UI 栈顶优先）；建造目录不可叠加在其他全屏 UI 上 |
| E17 | 鸡舍竣工当日 | 附赠 2 只鸡于竣工清晨生成在鸡舍内；当日即产蛋（让竣工日就有回报） |
| E18 | 同日下单多座建筑 | 允许并行多个工地（金钱与材料是真实约束，不再加人为排队） |

## 9. 经济与节奏自检（出厂前跑 headless 验证）

1. **Lv5 节点**：按宪法 M1 毕业线（200~350g/日）模拟，玩家应在 Lv5 后 7~10 游戏日内攒齐鸡舍全款全料；
2. **金币汇总量**：建造系统全清单总汇 ≈ 47,000g 金币 + 1,250 木 + 700 石（≈ 等效 57,000g），对照 Lv10 所需累计收入为合理的长线消费面；
3. **加工增值排序**：果酱（101g/槽日）> 干货（71g/槽日）> 直售，且加工棚总价 15 倍于烘干架——投入与回报单调一致；
4. **每个图纸竣工日必有可见回报**：鸡舍→当日产蛋（E17）、加工棚→当日可投料、温室→当日可耕种、农舍→当日换装+新房间；
5. 烘干架/加工槽工时只按游戏日推进——快进测试「下单后关页 7 真实天再打开」状态零变化。

## 10. 验收标准（对照宪法 §7 红线 + 本系统专项）

1. Lv3 玩家从按下 B 到放下第一段栅栏 ≤ 30 秒（教学成本验证）；
2. Lv5 玩家 2 分钟内完成「开目录→选鸡舍→放置→确认」，且当晚日结算屏出现「还差 1 天完工」（明日之诺供给验证）；
3. 任意离开（关页/切 tab）：工地、烘干、产蛋零推进且零恶化（宪法红线 3）；
4. 拆除任意装饰/设施 → 背包与金币精确恢复至建造前（refundRate=1.0 验证）；
5. 建造目录/确认框打开时 sim tick 完全停止（自动化测试断言 tick 计数）；
6. 全部建筑/设施贴图在 `assets/manifest.json` 可追溯至 CC0（宪法红线 5）；
7. 全流程键盘-only 与鼠标-only 各通关一遍（§4.5 等价性验证）；
8. 导入一份手工构造的重叠存档 → 触发 E12 收回补偿且不崩溃；
9. 建造系统全程未产生任何弹窗打断或与 HUD/quest 的耦合调用（宪法红线 4，代码层以 ESLint 边界规则看护）。

## 13. 待决问题（open questions）

- **Q1（修宪相关）**：宪法 M3 In 写「建筑 3 种」。本稿把农舍升级链、小型设施与装饰物归类为「设施/装饰 ≠ 建筑」纳入 M3——需总设计确认该口径；若不认可，农舍升级链与装饰物移出 M3 需走修宪流程标注去向；

- **Q3**：工匠职业对加工品售价的加成系数（本稿假设 +25%；SDV Artisan 为 +40%，因 Codestead 加工链更短建议调低）；

- **Q6**：温室解锁（Lv9）与换季规则启用（宪法「M3 后启用」）的时序匹配——若 M3 全程单季，温室价值打折，是否随温室竣工同步启用换季。
